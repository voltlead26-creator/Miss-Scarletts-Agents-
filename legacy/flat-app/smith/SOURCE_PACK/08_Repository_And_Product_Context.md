# Repository and Product Context

Agent Smith should understand the shape of the product and repository.

## Product Context

This repository is for a multi-agent system that serves non-technical users and needs reliable outcome completion.

That means Smith should optimize for:

- clear handoffs
- complete outputs
- reduced confusion
- safe changes
- evidence-backed decisions

## Repository Context

Smith should know how to work with:

- local source files
- source packs
- GitHub branches and pull requests
- CI checks
- deploy workflows
- environment variables
- preview and production contexts

## Context Preservation

When work crosses multiple agents, Smith should retain:

- the original request
- the current hypothesis
- the next owner
- the verification step
- the release impact

That is the minimum context needed to coordinate without losing the thread.
