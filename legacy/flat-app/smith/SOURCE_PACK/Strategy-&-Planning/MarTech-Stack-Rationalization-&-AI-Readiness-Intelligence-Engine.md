**Difficulty:** Advanced | **Time:** 25-30 min | **Tags:** martech, technology strategy, stack rationalization, AI readiness, marketing operations, budget optimization, cmo, b2b

## Overview

Conducts a full audit of your marketing technology stack, calculates per-tool ROI and utilization scores, identifies redundancy and consolidation opportunities, and outputs a prioritized AI-first modernization roadmap with vendor negotiation playbooks and projected savings. Use when you're under pressure to justify MarTech spend to the CFO, planning an AI transformation, or realizing your 40+ tool stack is creating more chaos than leverage.

- Expected cost savings or performance improvement

**Step 4 — Output the Rationalization Roadmap**
Produce:
1. A "Keep / Consolidate / Replace / Eliminate" verdict for every tool
2. A 12-month migration timeline with dependencies
3. Projected annual savings (conservative and aggressive scenario)
4. Top 3 vendor negotiation levers (renewals, consolidation discounts, multi-year deals)
5. AI tools to evaluate for adoption in next 90 days

**Scoring Definitions:**

Utilization Score:
- 9-10: Used daily by >80% of licensed seats, generating measurable output
- 7-8: Used weekly by majority of team, core to workflows
- 5-6: Used by some team members, results inconsistent
- 3-4: Occasional use, mostly by 1-2 people

Strategic Fit Score:
- 9-10: Directly enables a top-3 company strategic priority
- 7-8: Supports important marketing function but not top priority
- 5-6: Useful but not aligned to current priorities
- 3-4: Legacy tool retained for historical reasons
- 1-2: Cannot articulate why we have it

- 1-2: Requires human judgment; AI augmentation only

MODULE 2: REDUNDANCY & CONSOLIDATION ANALYSIS

**Redundancy Detection Framework:**

For each functional capability, identify every tool in your stack that performs it:

| Capability | Primary Tool | Secondary Tool(s) | Annual Cost Overlap | Redundancy Type |
|---|---|---|---|---|

Redundancy Types:
- Full Redundancy: Both tools do the same job (eliminate one immediately)
- Partial Overlap: 60-80% functional overlap (consolidate to one with feature negotiation)
- Capability Creep: A tool you already own has this feature you're paying separately for
- Integration Redundancy: Two tools purchased to solve a data connectivity problem a native integration now solves free

**Top Consolidation Opportunities (prioritized by savings):**

For each consolidation opportunity, output:
- Current state: Tool A ($X/yr) + Tool B ($Y/yr) = $Z total
- Recommended state: Tool C (or Tool A with Tool B eliminated)
- Migration complexity: 1-10 (1 = flip a switch, 10 = 6-month migration)
- Annual savings: $X conservative, $Y aggressive
- Risk: What capability is lost, and does it matter?

**Hidden Cost Calculation:**

Most MarTech audits only look at license fees. Calculate total cost of ownership:

| Tool | License Cost | Integration/Dev Hours ($/yr) | Training Hours ($/yr) | Admin Hours ($/yr) | Support Costs | True TCO |
|---|---|---|---|---|---|---|

Rule of thumb: For complex tools (MAP, CDP, analytics platforms), actual TCO is typically 2.5-4x the license cost. Flag any tool where TCO exceeds license by >3x AND utilization score is below 6.

## Integration Tips

**HubSpot (if consolidating to HubSpot as primary platform):**
- Before canceling any tool, export: all contacts/segments, email templates, workflow logic, historical performance data, and custom field mappings
- Use HubSpot's Migration Tool for contact/company data; rebuild workflows manually (don't import old MAP logic 1:1 — redesign for current state)
- Create a HubSpot Property: `martech_source` on Contact to track which tools attributed to each record during transition
- Enable HubSpot's AI features before rationalizing: Content Assistant, AI Email Writer, ChatSpot — run for 30 days to confirm capability before eliminating content AI subscriptions

**Salesforce (CRM of record):**
- Before eliminating any integrated tool, map all Salesforce → [Tool] and [Tool] → Salesforce data flows in a spreadsheet
- Flag any tool with a managed package installed in Salesforce — these require IT coordination to safely remove without breaking fields or reports
- Use Salesforce AppExchange health check to identify unused app connections consuming API limits

**Notion / Confluence (documentation layer):**
- Build a MarTech Stack Registry in Notion: one page per tool with: owner, renewal date, cost, use case, key contacts (vendor CSM), integration map, and SLA for uptime
- Link to this registry in every quarterly marketing ops review — it becomes a living document, not a one-time audit
- Add renewal reminders 90, 60, and 30 days before each contract end date

**Zapier / Make (automation rationalization):**
- Audit existing Zaps/workflows before eliminating tools — any Zap that connects a tool you're removing must be rebuilt or deactivated before canceling the subscription or you'll generate silent failures
- Export workflow documentation from Make/Zapier as part of the audit output
- When consolidating, look for native integrations first (HubSpot ↔ Salesforce native connector eliminates 3-5 Zaps for most teams)

**Procurement / Finance Integration:**
- Export all contract end dates into your company's procurement calendar immediately
- For tools with auto-renew clauses, calendar the opt-out deadline (typically 30-60 days before renewal) — this is where most wasted spend lives
- Build a shared Google Sheet with: Tool | Vendor | Contract End Date | Auto-Renew Opt-Out Deadline | Owner | Action Required

## Troubleshooting

**Problem: Stakeholders resist tool consolidation because "their team uses it and loves it"**
Cause: Political ownership of tools; individual teams have budget authority or emotional attachment to specific platforms; consolidation feels like losing resources.
Fix: Reframe consolidation as capability upgrade, not cost-cutting. Run a structured "capability gap analysis" for any tool being eliminated: list all features the team uses, map each feature to the consolidation target, and document gaps. If a gap is real (not imagined), either (a) negotiate that feature into the consolidation target's contract, or (b) use the gap as justification to keep the tool. Most gaps turn out to be imagined or rarely used features that teams remember using but actually don't. Presenting data (utilization analytics from the vendor's admin console) neutralizes emotional attachment.

**Problem: The vendor won't provide utilization data during negotiation prep**
Cause: Vendors know that low utilization = cancellation risk; they deliberately obscure usage metrics in their dashboards or require admin-level access that the negotiating team doesn't have.
Fix: Pull usage data from your own systems before vendor conversations. For email tools, pull send volume and active workflow counts from your MAP admin. For analytics tools, pull login frequency from your SSO provider (Okta, Google Workspace admin). For any tool with an API, use Zapier or Clay to pull event data for the last 90 days and calculate actual usage. Come to every renewal with your own utilization data — don't ask the vendor for theirs.

**Problem: Migration timeline slips, causing overlap in subscriptions and double-paying for both tools**
Cause: Underestimated data migration complexity; IT or engineering dependencies not flagged early; team resistance to changing workflows mid-quarter.
Fix: Apply the "Overlap Budget" rule: when projecting migration costs, add 2 months of dual subscription cost to your migration budget as a contingency. Build migrations around contract end dates, not around convenience — if migration to HubSpot isn't complete 60 days before Marketo renewal, either push for a month-to-month extension from Marketo or commit the resources to complete migration. Never auto-renew a tool you're actively migrating away from — set a hard deadline and escalate to CMO if engineering resources aren't allocated.
