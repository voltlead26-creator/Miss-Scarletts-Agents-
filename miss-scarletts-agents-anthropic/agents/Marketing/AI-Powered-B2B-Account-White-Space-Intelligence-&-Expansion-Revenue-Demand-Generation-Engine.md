## Integration Tips

**Salesforce + HubSpot:**
- Sync ERS as a custom field on the Account object; build Expansion Pipeline as a custom opportunity type with "Expansion" stage tracking
- Create a Salesforce flow that auto-creates an expansion task for the AE/CSM when ERS crosses threshold; include signal summary in task description
- Build a HubSpot workflow that auto-enrolls contacts into expansion sequences when Salesforce account ERS updates via Zapier or HubSpot-Salesforce native sync

**Amplitude / Mixpanel / Pendo:**
- Create a dedicated "Expansion Signals" dashboard with all weighted signals visible at account level
- Set up behavioral cohorts for each expansion motion so you can measure sequence performance against baseline adoption cohorts
- Use Pendo in-app guides triggered by specific feature-usage events (e.g., when a user hits the API call limit, show upgrade prompt with specific tier comparison)

**Customer.io / Iterable:**
- Map product events as Customer.io "events" and use event-triggered campaigns for real-time sequence enrollment — ensures trigger fires within minutes of qualifying behavior
- Use liquid templating in Customer.io to dynamically insert product usage data (e.g., "You've run 47 reports this month — Professional users average 180") directly from your data warehouse via Segment or direct API

**Gainsight / Totango:**
- Create a "Marketing Expansion Assist" playbook that auto-launches when ERS crosses 75 — assigns outreach task to CSM while email sequence runs in parallel
- Feed expansion campaign engagement data (email clicks, page visits) back into Gainsight health score so CS sees the full customer engagement picture

**Snowflake / BigQuery:**
- Build the ERS model as a dbt model that refreshes daily, outputs to a `marketing_expansion_scores` table, and syncs to Salesforce and Customer.io via Hightouch or Census
- Create expansion attribution tables that join campaign touch data with contract expansion events for true multi-touch expansion attribution

## Troubleshooting

**Problem:** Expansion email open rates are low (<20%) despite good product usage signals.
*Solution:* The subject line is likely too generic or lacks personalization. Insert a specific product usage data point in the subject line (e.g., "You've used 3 of 12 available features this month"). Also audit your send time — expansion emails perform best Tuesday–Thursday, 9–11am in the recipient's local timezone. Check if your customer email domain has engaged/non-engaged segments and suppress non-openers after 60 days to protect sender reputation.

**Problem:** The Expansion Readiness Score is identifying many "ready" accounts that don't convert when sequenced.
*Solution:* Your score weights are likely over-indexing on passive signals (page visits, feature activations) vs. active intent signals (pricing page visits, upgrade CTA clicks, self-serve checkout initiated). Recalibrate: add a 20-point multiplier for any pricing/upgrade page visit in last 30 days and remove or reduce weight for passive adoption signals. Also check if the accounts flagging as "ready" are actually the economic buyers or just end-users — if your champion doesn't control budget, no amount of email will close the expansion.

**Problem:** CS team is frustrated that marketing expansion campaigns are reaching out to "their" accounts without coordination.
*Solution:* This is a process and trust issue, not a tech issue. Fix with a tiered-intervention model: accounts <$25K ACV run fully automated (marketing owns); accounts $25K–$100K use marketing email + CS alert at engagement threshold (joint ownership); accounts >$100K suppress from automated sequences and require CS-initiated expansion motion. Present CS with weekly "expansion-ready account" reports so they feel informed and in control. Run a 30-day test showing CS that marketing-sequenced accounts have higher expansion conversion rates to build buy-in.
