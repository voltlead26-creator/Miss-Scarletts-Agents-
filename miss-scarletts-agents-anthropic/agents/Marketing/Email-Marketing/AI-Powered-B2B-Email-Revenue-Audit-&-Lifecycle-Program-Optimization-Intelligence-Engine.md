## Integration Tips

- **HubSpot**: Run the audit output directly into HubSpot's Email Health dashboard (Marketing > Email > Health). Cross-reference AI-identified leakage points with HubSpot's native "Hard bounce," "Unsubscribed," and "Marked as spam" filters. Use Workflow Enrollment History to validate sequence logic gaps.
- **Marketo**: Export Program Performance Report for all active email programs. Feed the data table directly into the Advanced Version's Context section. Use Marketo's Deliverability Tools (Email Insights) to validate deliverability scores before and after list suppression.
- **Pardot/MCAE**: Pull Engagement Studio reports for each active path. Combine with Salesforce Campaign Influence data to validate email-attributed pipeline figures. Use Prospect Lifecycle Report to identify lifecycle coverage gaps.
- **Google Sheets**: Build the RPE (Revenue Per Email) tracking dashboard using this formula: `=IFERROR(Pipeline_Attributed/Emails_Sent,0)`. Track weekly by sequence to identify decaying programs before they become a crisis.
- **Zapier + Notion**: Auto-log weekly email performance metrics into a Notion database using Zapier (HubSpot → Zapier → Notion). Run a monthly audit prompt refresh to catch new leakage points as your program evolves.
- **Salesforce Campaign Attribution**: Map each email sequence to a Salesforce Campaign with Type = "Email." Use Campaign Influence with Custom Attribution Model to isolate email contribution within multi-touch deals.

## Troubleshooting

**Issue: The AI audit scores are generating generic recommendations instead of program-specific fixes.**
*Solution*: The most common cause is insufficient data in the Context section. Ensure you include actual open/click/bounce/complaint numbers, a complete sequence list with current conversion rates, and at least one known problem. The more specific your metrics, the more surgical the output. If running the Quick Version, manually add your top 3 performing and bottom 3 performing emails by name before submitting.

**Issue: Projected revenue recovery estimates seem unrealistically high.**
*Solution*: The Advanced Version's constraints section instructs the AI to use 20th percentile improvement assumptions. If estimates still seem high, add this line to the Constraints section: "Use 10th percentile improvement assumptions and always state the confidence interval." For board presentations, run the audit twice — once conservative, once moderate — and present the range.

**Issue: Rewritten emails don't match our brand voice.**
*Solution*: Add a Brand Voice section to the Context block before Deliverables: "Brand Voice: [3 adjectives] — [1 sentence description of tone]. Examples of our voice: [paste 2-3 sentences from your best-performing existing email]." Alternatively, run the rewrite output through your Executive Ghostwriting prompt to apply brand voice as a second pass.
