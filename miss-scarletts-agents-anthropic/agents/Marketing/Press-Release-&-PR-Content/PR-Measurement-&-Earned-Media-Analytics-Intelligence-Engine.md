Use this when you need to prove PR ROI to the CFO, defend the communications budget, or optimize your media strategy for coverage that actually converts.

## Integration Tips

- **Meltwater / Cision / Mention**: Export coverage reports as CSV, paste directly into the prompt. The AI will parse and score each placement automatically.
- **Google Analytics 4**: Export sessions by date with source/medium breakdown. Cross-reference major coverage dates against traffic spikes using the Explorations > Funnel Exploration report.
- **Salesforce / HubSpot**: Add "Earned Media" as a Lead Source option and create a field for "Press Coverage Cited" in discovery call notes. Use this to build a proper closed-loop attribution dataset over 2-3 quarters.
- **Notion / Google Slides**: Paste the executive dashboard output directly into your board deck or QBR template. The table format renders cleanly in both tools.
- **Make (formerly Integromat) / Zapier**: Automate weekly coverage pulls from Mention or Brand24 → feed into a structured Google Sheet → run this prompt monthly to generate rolling reports without manual data entry.
- **Airtable**: Build a coverage tracking database with fields matching this prompt's schema. Connect via Zapier to auto-populate when new coverage is detected, making the prompt run faster with pre-structured input.

## Troubleshooting

**Problem:** "I don't have structured coverage data — just a list of article URLs."
**Solution:** Paste the URLs with publication name and date. Ask the AI to visit each URL (if you're using a tool with web access) or manually provide headline, sentiment (positive/neutral/negative), and whether you were the focus or just mentioned. Even rough data produces directionally useful QAEMV and SOV calculations.

**Problem:** "The pipeline attribution numbers feel made-up / the CFO won't buy them."
**Solution:** Use the prompt's confidence-level flags (High/Medium/Low). Present "influenced pipeline" separately from "directly attributed pipeline." Anchor the modeled numbers in observable data (e.g., "demo requests were 47% above baseline in the 72 hours following TechCrunch coverage"). Over 2-3 quarters, the correlation data becomes statistically defensible.

**Problem:** "Competitors have much more coverage — the SOV numbers are discouraging."
**Solution:** Reframe SOV by narrative category instead of total volume. A startup can own 80% SOV on a specific theme (e.g., "AI ROI in procurement") while having 10% total SOV. Use the Narrative Ownership Map to find the white space where you can dominate rather than competing on volume against established players.
