## Integration Tips

- **Gainsight**: Load the Adoption Scoring Model (Module 1.1) directly into Gainsight's Success Plans as a CTAs scorecard. Use Gainsight Journeys to automate Track A and B sequences with usage-data branching.
- **Pendo**: Map in-app tooltips (Module 3.1) and modals (Module 3.2) to Pendo Guides. Use Pendo's NPS + feature flags to trigger Track A Day 1 checklist.
- **Intercom**: Use Intercom Series to orchestrate multi-step email + in-app message sequences. Connect Intercom Events to product usage milestones for behavioral triggers.
- **HubSpot**: Sync adoption health scores to HubSpot contact properties. Use HubSpot Workflows with Score property triggers to route to CS sequences vs. automated nurture.
- **Salesforce**: Push expansion signals (Module 5.1) as Salesforce Tasks to the owning CSM. Map adoption score to a custom field on the Account object for CS and AE visibility.
- **Notion/Google Slides**: Use the EBR ROI Report Generator (Module 5.3) to auto-populate a Notion template or Google Slides deck via API before each QBR.
- **Zapier/Make**: Connect product database (feature usage events) → score calculation → CRM update → sequence enrollment in a fully automated Zapier workflow. No engineering resources required.

## Troubleshooting

**Problem**: Adoption sequences get high open rates but low activation (customers read but don't act)
**Fix**: The barrier is likely ability, not motivation. Shorten the path to first use — replace "click to learn more" CTAs with deep-links directly into the feature with a pre-populated example from the customer's own data. Add a 2-minute Loom video showing the exact first 3 clicks.

**Problem**: Customers report being overwhelmed by too many adoption messages
**Fix**: Implement a global message frequency cap (max 2 adoption touchpoints per week per customer) and suppress sequences for customers currently in a support escalation. Prioritize sequences by feature business impact — only run the highest-value feature campaign per customer at a time.

**Problem**: CSMs aren't using the escalation signals — they're missing CTA alerts
**Fix**: The issue is CTA fatigue — CSMs get 30+ Gainsight CTAs and ignore them. Consolidate adoption CTAs into a single "Weekly Adoption Digest" Slack notification per CSM (not per account), ranked by urgency score. Limit to top 5 accounts needing attention per week.
