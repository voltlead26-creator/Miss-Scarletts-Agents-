## Integration Tips

**HubSpot / Salesforce (CRM as consolidation anchor):**
Export this report's recommended stack architecture as a CRM property mapping document. Use the integration dependency map to rebuild CRM-to-MAP data flows before retiring any tool. Run a data audit in Salesforce before migration: missing/null fields reveal hidden tool dependencies you didn't document.

**Snowflake / BigQuery (Data Warehouse):**
For tools with native data warehouse connectors (Looker, Sigma, dbt), migrate analytics tool LAST — not first. Your warehouse is the source of truth; validate all data flows into it before sunsetting any pipeline tool. Use the consolidation as an opportunity to standardize UTM parameters and campaign naming conventions across the new stack.

**Zapier / Make (Workflow Automation):**
Before retiring any tool, export all active Zap/Make workflows that touch it. The consolidation AI should flag all automation dependencies as a migration pre-requisite. Build a "workflow inventory" document from Zapier's API before wave 1 begins.

**Notion / Confluence (Documentation):**
Store the full consolidation report, tool assessment cards, and migration runbooks in a shared wiki. Tag each tool with its deprecation date and replacement. Create a "MarTech single source of truth" page linked from your team wiki's top nav.

**Vendor Negotiation (direct):**
Use the AI-generated comparison tables as leverage in renewal conversations. Share the "AI-native alternative" section with your current vendor during renewal discussions — most vendors will offer 20-40% discounts when they know you have a credible alternative evaluated.

**CFO/Finance System Integration:**
Export the financial model as a Google Sheet or Excel template. Format savings projections to match your company's operating model (monthly vs. annual, GAAP vs. cash basis). Link MarTech cost line items to your finance team's cost center codes so savings flow directly into budget tracking.

## Troubleshooting

**Issue: The AI doesn't know current vendor pricing, so cost estimates are inaccurate.**
Solution: Before running the prompt, pull actual invoice amounts from your finance system or vendor portals. Use the prompt to analyze structure and redundancy, then manually input real contract values. For annual contracts, calculate monthly equivalent and note renewal date — these are the most valuable inputs for wave sequencing.

**Issue: The consolidation roadmap doesn't account for our sales team's dependency on a specific tool.**
Solution: Add a "Sales-Critical Tool" flag to your inventory table for any tool where sales reps are the primary user (not marketing). The prompt will then protect those tools from wave 1 and escalate them to stakeholder playbook discussions. Run the prompt again with explicit "Cannot replace" constraints for those tools.

**Issue: The AI recommends retiring a tool our team loves, creating adoption resistance.**
Solution: Include the "Team NPS" column in your tool inventory table. The AI will weight team sentiment in the risk register and generate targeted change management language for each tool retirement. For high-NPS tools marked for consolidation, ask the AI to generate a "feature parity proof" document — showing team members that the replacement tool covers 95%+ of their workflows — before the transition begins.
