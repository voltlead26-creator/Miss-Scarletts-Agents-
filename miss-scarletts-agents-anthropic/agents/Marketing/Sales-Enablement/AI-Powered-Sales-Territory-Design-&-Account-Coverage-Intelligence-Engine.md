## Integration Tips
- **Salesforce**: Use Territory Management 2.0 (Enterprise/Unlimited edition) to mirror this model — create custom territory rules using Account fields (Industry, AnnualRevenue, NumberOfEmployees). Export output tables directly to the Territory Assignment Rules setup.
- **HubSpot**: Use the Territory feature under Sales Hub Enterprise. Map firmographic scoring to custom contact/company properties; use Workflows to auto-assign accounts to owners based on ICP criteria.
- **Clari / Gong**: Feed territory structure into Clari's capacity model for real-time quota attainment forecasting. Connect territory data to Gong Engage for SDR sequence assignment per territory.
- **Google Sheets / Notion**: Paste the territory carving table into a shared planning doc for AE sign-off. Link to a live Salesforce report for ongoing monitoring.
- **ZoomInfo / Cognism / Apollo**: Use the ICP firmographic thresholds from Section 1 to build named account lists by territory. Export to CSV and import into CRM as the "Territory Whitespace" list.
- **Zapier / Make**: Automate territory assignment: when a new account is created in CRM matching firmographic rules, auto-assign to the correct AE territory owner and enroll in the relevant SDR sequence.

## Troubleshooting
- **Problem:** AEs resist the new territory structure, claiming their accounts are being taken away.
  **Solution:** Run the territory balance analysis transparently — show each AE their new TAR vs. old TAR. If an AE loses accounts, compensate with a higher proportion of whitespace (unworked) accounts. Conduct 1:1 handoff reviews. Implement a 90-day "transition credit" policy where reps still receive partial commission on accounts they owned during the prior period.

- **Problem:** The quota model produces quotas that finance says are too low vs. company ARR target.
  **Solution:** Add an over-quota buffer (110-125% of company target is standard). Check if win rate or ACV inputs are based on trailing-twelve-month actuals vs. aspirational targets — ensure you're modeling from actuals. Also confirm pipeline coverage multiple is set conservatively (4-5× for mid-market, 5-6× for enterprise).

- **Problem:** CRM data quality is too poor to run accurate territory analysis (missing industries, no revenue data, wrong headcount).
  **Solution:** Before running this prompt, run a data enrichment pass using ZoomInfo, Clearbit, or Cognism to append firmographic data to all CRM accounts. At minimum, enrich the top 2,000 accounts by last-activity date.
