# Net Revenue Retention (NRR) Forecasting & Cohort Expansion Intelligence Engine - AI-Powered Portfolio-Level Retention Economics

**Difficulty:** Advanced | **Time:** 25-30 min | **Tags:** NRR, GRR, net revenue retention, cohort analysis, expansion revenue, churn, SaaS metrics, revenue forecasting, customer success, B2B SaaS

## Overview
Analyzes your full customer portfolio using cohort-level retention data to diagnose what's driving NRR above or below benchmark, forecast future NRR trajectory, and generate a prioritized intervention roadmap to systematically move from 100% toward 120%+ NRR. Use this when your board is asking why NRR is declining, you're preparing for a fundraise, or you need to build a data-driven plan to convert your CS organization into a net revenue growth engine — not just a churn prevention function.

## Integration Tips

**Gainsight:**
- Build a custom NRR Dashboard using Gainsight's "Company" object — pull Expansion ARR, Contraction ARR, and Churned ARR fields into a portfolio-level dashboard filterable by segment, CSM, and cohort month
- Create a "NRR Risk" rule: Any account where contraction probability (via health score trend) exceeds 60% → auto-create a Gainsight Call to Action (CTA) assigned to the CSM with a 7-day SLA
- Use Gainsight Timeline to log "Expansion Qualified" events — feed these into a custom report showing expansion pipeline by segment and quarter

**Salesforce + RevOps:**
- Add three custom Opportunity fields: Expansion ARR, Contraction ARR, Churned ARR — typed as currency. This enables proper NRR waterfall reporting in any BI tool connected to Salesforce
- Build a Salesforce report type "NRR Waterfall by Segment" using Account Segment + Opportunity Type (New / Expansion / Contraction / Churn) + Close Date — run monthly and export to your board deck automatically via Salesforce Reports Scheduled Runs
- Create an Account-level field "Expansion Score" (from Upsell/Cross-Sell Intelligence Engine) that feeds into a "Top Expansion Opportunities" list view for AEs and CSMs

**HubSpot:**
- Use custom properties: "ARR at Start," "ARR at End," "Expansion ARR," "Contraction ARR," "Churned ARR" on the Company object. Build a calculated property for NRR: (End ARR / Start ARR) × 100
- Create a HubSpot Dashboard "NRR Command Center" with: (1) Current NRR gauge, (2) Expansion ARR trend line (weekly), (3) Churn ARR by reason (using deal lost reason field), (4) Cohort table showing 6/12/24-month retention by signup quarter

**Looker / Tableau / Mode (for BI teams):**
- Build a Cohort Retention Matrix: X-axis = months since first close, Y-axis = cohort quarter, cells = % of original ARR retained. This is the single most powerful NRR diagnostic visualization — it instantly shows where decay occurs and whether expansion is compounding
- Layer expansion ARR onto the cohort matrix: Where cells exceed 100%, the cohort is NRR-positive. Identify the "crossing point" (month when cohort turns NRR-positive) and track it over time — a declining crossing point means your expansion motion is improving

**Notion (for CS Ops teams without heavy BI tooling):**
- Create a Notion database "NRR Tracker" with fields: Customer Name, ARR at Period Start, Current ARR, Expansion ARR (12M), Contraction ARR (12M), Churn Date (if applicable), Account NRR, Segment. Sort by Account NRR ascending to instantly see your worst-performing accounts
- Automate data pull via Zapier: HubSpot deal close → update Notion row → trigger Slack notification to CS lead when account NRR drops below 90%

## Troubleshooting

**Problem: You don't have clean cohort data — only know current ARR and churn rate**
Fix: Use the Quick Copy-Paste version with just total portfolio numbers. The engine will estimate cohort curves from available signals and flag assumptions clearly. Even without precise cohort data, the waterfall decomposition and intervention prioritization are fully actionable. For future cohorts, start tagging customers in CRM with "Close Month" (e.g., 2025-Q1) immediately — you'll have clean cohort data in 6 months.

**Problem: NRR looks healthy (>110%) but the board is still asking about retention risk**
Fix: Run Module 3 (Segment-level NRR diagnosis) — portfolio NRR often masks a dangerous SMB churn rate being offset by Enterprise expansion. A 115% overall NRR with 80% GRR in SMB is a ticking clock; once Enterprise logo churn increases, the whole portfolio tips. The prompt will surface this concentration risk automatically if you provide segment breakdowns.

**Problem: The intervention roadmap shows programs that require 6+ months to implement — you need a quick win for next board meeting**
Fix: Prompt the Advanced version and add this instruction at the end: "For each intervention, also generate a 30-day tactical sprint version — the minimum viable version that creates a measurable signal within one month without requiring new tooling or headcount." This forces the model to separate program design from quick-start execution.
