## Overview
Transforms raw customer conversation data — call transcripts, support tickets, email threads, chat logs, and NPS verbatims — into actionable intelligence that predicts churn, surfaces expansion opportunities, and auto-generates CS response playbooks before accounts deteriorate. Use it when you need to move beyond lagging health scores and act on the *qualitative* signals that numbers miss.

### SECTION 3: ANALYSIS FRAMEWORK

**A. Multi-Dimensional Sentiment Scoring**

For each data source provided, apply the EMPATHY scoring model:
- **E**motion intensity (1–5 scale per statement)
- **M**omentum direction (positive arc / negative arc / volatile)
- **P**erson-level differentiation (champion vs. economic buyer vs. end user sentiment — these often diverge)
- **A**ction language analysis (passive vs. active, committed vs. hedging)
- **T**one shift detection (where in the conversation did sentiment change?)
- **H**idden dissatisfaction signals (polite complaints, feature requests masking deeper frustration, "we'll see" language)
- **Y**our product vs. competition framing (how they talk about alternatives)

Output: Overall account sentiment score (1–10), per-stakeholder sentiment breakdown, and a sentence-level emotion heatmap identifying the 5 most critical moments.

**B. Churn Risk Signal Extraction (SCORE Framework)**

Extract every instance of:

| Signal | Examples to Flag | Severity Weighting |
|--------|-----------------|-------------------|
| **S**atisfaction gaps | "it doesn't do X", "we expected Y", "this is taking too long" | Medium–High |
| **C**ompetitive mentions | Any competitor named, "we're evaluating alternatives", "our board asked about X" | High–Critical |
| **O**wnership changes | Champion departure, new executive, re-org, acquisition mention | High |
| **R**OI doubts | "can't justify the cost", "hard to show value internally", "our CFO is asking" | High–Critical |
| **E**scalation language | "I need to escalate", "this is unacceptable", "executive involvement", legal language | Critical |

For each signal: quote it verbatim, rate severity, identify which stakeholder said it, and prescribe the specific CS action required within 48 hours.

**C. Expansion Opportunity Extraction (GROW Framework)**

Map every positive signal:

- **G**rowth use cases mentioned: new teams, new workflows, expanded scope
- **R**OI articulation: customer quantifying value, success stories, case study potential
- **O**ther buyer signals: mentions of budget availability, new headcount, strategic initiatives
- **W**illingness indicators: "we'd love to", "can you help us with", "what does it cost to add"

For each opportunity: estimate expansion ACV potential (Low: <$10K, Medium: $10–50K, High: >$50K), identify the right person to pitch, and generate a 2-sentence expansion opener.

**D. Automated Response Playbook Generation**

Based on detected sentiment and risk level, generate:

**IMMEDIATE (0–24 hours)**:
- Internal Slack/Gainsight alert message (for CSM team)
- Customer-facing email (subject line + full body — personalized to detected tone and specific issues raised)
- If escalation warranted: executive briefing summary (5 bullet points, < 150 words)

**SHORT-TERM (1–7 days)**:
- Discovery call agenda focused on surfacing and resolving identified risks
- 3 questions the CSM must ask to validate or dismiss the churn risk
- Proof point or case study to deploy for ROI reinforcement

**STRATEGIC (30 days)**:
- Success plan adjustment recommendations
- Executive sponsor re-engagement strategy
- Renewal negotiation preparation notes (if renewal is within 90 days)

## Integration Tips

- **Gong/Chorus**: Export call transcript JSON via API → parse with Claude → pipe output to Gainsight account notes and auto-create CS tasks
- **Gainsight/Totango**: Map extracted SCORE signals to Gainsight risk fields; trigger automated playbooks via Gainsight Rules Engine using risk severity output
- **Salesforce**: Write revenue risk quantification output to Opportunity "Churn Risk" field; auto-assign follow-up tasks to CSM via Process Builder
- **Zendesk/Intercom**: Run ticket thread analysis on a scheduled basis (weekly); flag tickets scoring High+ risk for CSM review queue; route Critical escalations to CS Manager Slack channel via Zapier webhook
- **HubSpot**: Use conversation outputs to update Contact "Sentiment" property and trigger CS email sequences; log analysis summaries to Deal timeline
- **Slack Integration**: Automate daily digest: pull Gong transcripts + new tickets → run analysis → post risk summary to #cs-alerts channel with @mention of CSM
- **Notion/Confluence**: Auto-generate account intelligence pages with weekly sentiment trend data; build a searchable "risk signal library" from recurring patterns
- **Zapier/Make.com**: Trigger analysis pipeline: New Gong call completed → Extract transcript → Claude analysis → Update CRM + notify CSM + file in Notion

## Troubleshooting

**Problem**: Sentiment analysis is too generic — every account flagged as "Medium risk" regardless of actual signals.
**Solution**: Increase specificity in Section 2 inputs. Include verbatim quotes rather than summaries. The EMPATHY and SCORE frameworks require exact language to detect hedging, tone shifts, and competitive references — paraphrased summaries strip out the signal-bearing words.

**Problem**: The generated email feels robotic or doesn't match the customer relationship tone.
**Solution**: Add a "Relationship Context" field in Section 1: describe the CSM-customer relationship history (new account, 3-year relationship, previous escalation, etc.) and the customer's communication style (formal/informal, data-driven/relationship-driven). Include 1–2 sample sentences from past emails to calibrate tone.

**Problem**: Can't run cross-account pattern analysis — too much data for a single prompt.
**Solution**: Run individual account analyses first, then create a synthesis prompt: paste the 5–10 Executive Risk Summaries into a new Claude session and ask it to identify recurring themes, common product complaints, and competitor patterns. Use Claude Projects or a document store to maintain analysis history.
