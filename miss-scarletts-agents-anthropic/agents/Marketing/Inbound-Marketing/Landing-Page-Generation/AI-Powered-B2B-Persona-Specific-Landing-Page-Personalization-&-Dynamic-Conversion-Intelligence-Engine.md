## Integration Tips

- **HubSpot CMS Hub**: Use Smart Content rules to serve persona-specific content blocks based on contact properties (lifecycle stage, job title from form, or Clearbit enrichment). Set up `persona_variant` custom contact property to track which page version generated each lead.
- **Unbounce / Instapage**: Use Dynamic Text Replacement (DTR) for headline personalization via UTM parameters. Create separate pages per persona using the base template, then clone and customize — maintain one design system in the master template.
- **Mutiny / Intellimize**: For programmatic personalization on high-traffic sites (>50K sessions/month), use these platforms to run AI-driven personalization at scale without manually building N variants per persona — they handle firmographic routing and multi-variant testing automatically.
- **Clearbit Reveal / 6sense / Demandbase**: Install the IP enrichment JavaScript tag on all landing pages. Pass enriched firmographic data (company, industry, revenue, title inferred from IP) into HubSpot contact properties and use for variant routing and nurture sequence assignment.
- **Google Analytics 4**: Create custom dimensions for `persona_variant` and `routing_method` (UTM vs. firmographic vs. default). Build a GA4 Exploration report comparing conversion rates, engagement time, and scroll depth by persona variant — use this for weekly optimization decisions.
- **Salesforce**: Create a custom field `Landing_Page_Persona_Variant__c` on the Lead object, populated via HubSpot sync or direct form-to-SFDC integration. Use this field to route leads to the right SDR queue and pre-populate the SDR's outreach sequence with persona-matched email templates.
- **Zapier / Make**: Automate the workflow: form submission → add to HubSpot list based on persona variant → trigger persona-matched email sequence → notify SDR in Slack with persona context card.

## Troubleshooting

**Problem: Low traffic volume prevents reaching statistical significance in A/B tests**
Solution: Merge low-traffic persona variants using rule-based dynamic content blocks rather than separate pages — run multivariate testing across combined segments. For segments with <200 sessions/month, use a revenue-weighted decision framework instead of pure statistical testing: if the persona-matched variant consistently produces higher-ACV deals (even from a small sample), implement it as the default while continuing to collect data. Also consider using Bayesian testing tools (like VWO's Bayesian engine) which reach actionable conclusions faster than frequentist approaches for small-sample B2B environments.

**Problem: Clearbit/6sense IP enrichment identifies <40% of visitors, leaving most traffic in the unidentified "default" variant**
Solution: Build a smarter default variant by analyzing which persona's messaging historically converts best for unidentified traffic (usually the economic buyer/CFO variant for enterprise products, as it addresses ROI — a universal concern). Additionally, use progressive profiling: add a single qualifying question to your form ("What's your primary role?") with options that map to persona variants — use the answer to assign the contact to the right persona bucket AND to serve a confirmation page experience matched to their selection.

**Problem: Sales team reports that leads from persona-targeted pages have mismatched expectations (e.g., VP Ops who booked a demo expecting an operations-focused conversation but gets a generic demo)**
Solution: Pass the `persona_variant` field from HubSpot into your meeting booking tool (Chili Piper, Calendly) and use it to: (a) route to the SDR/AE with the strongest persona-specific expertise, (b) auto-populate the pre-meeting prep email with persona context ("Your prospect is a VP Operations focused on contract processing efficiency — lead with the Monday morning summary feature"), and (c) trigger a persona-specific demo script in Salesforce/HubSpot for the rep to follow. This closes the personalization loop from ad click through to closed deal.
