## Integration Tips
- **Stripe / Chargebee / Recurly:** Export pricing tier performance data to feed into hypothesis prioritization — identify which tier has the lowest conversion rate and prioritize objection neutralization copy there.
- **Hotjar / FullStory / Microsoft Clarity:** Run heatmap and session recording on the pricing page for 2 weeks before running the CRO prompt; use scroll depth and rage-click data to identify friction points, then feed those observations into the "top objections" field.
- **HubSpot / Salesforce:** Pull "lost deal" reason data filtered by "price" or "competitor" from the past 90 days and paste as the custom objection input — the prompt will generate inline page copy that directly addresses what's killing deals.
- **Optimizely / VWO:** The A/B test roadmap output maps directly to experiment creation in these tools; each hypothesis includes the metrics to configure as primary/secondary goals.
- **Figma:** Paste the "Designer Notes" callout blocks directly into a Figma page brief — each section includes layout directives (e.g., "place metric callouts directly below the hero CTA, not in the footer").
- **Notion / Confluence:** Use the full output as a living pricing page CRO wiki that your product, marketing, and sales teams can reference when making copy or positioning changes.

## Troubleshooting
- **The copy feels generic or feature-focused, not ROI-focused:** This usually means the ICP description was too vague. Add the buyer's specific job-to-be-done (e.g., "Engineering managers who are accountable for sprint velocity and release date commitments") and re-run the Advanced version.
- **The A/B test hypotheses overlap with each other:** Prioritize tests that change only one variable at a time. If multiple hypotheses target the same element (e.g., CTA button copy), stack them sequentially — run test 1 to completion before starting test 2 — and ask the prompt to re-sequence so high-traffic elements are tested first.
- **The tier architecture recommendations don't match your business model (e.g., usage-based pricing):** Add a "Pricing model context" field to the Advanced version specifying your model (e.g., "usage-based: $0.01 per API call with $25/month minimum"), and note that decoy-effect tier recommendations should be adapted to create usage thresholds rather than feature gates.
