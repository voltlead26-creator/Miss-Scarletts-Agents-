## Overview
This engine continuously monitors competitor pricing changes, analyzes win/loss patterns tied to pricing, and auto-generates sales enablement responses, messaging pivots, and deal-saving playbooks when competitive pricing pressure is detected. Use it when a competitor cuts prices, launches a new tier, or when deals are stalling on price objections.

## Troubleshooting

**Problem: Sales reps revert to discounting instead of using the value playbook**
Solution: The playbook needs to be shorter, not longer. Distill to a 1-page "Quick Reference Card" with only 3 talk tracks. Run a 20-minute Gong call review session showing a winning rep using value reframe. Tie competitive pricing win rate to rep scorecard (not just quota attainment). If discounting is happening, set a hard approval gate at >10% discount requiring manager sign-off with a required note: "Why value playbook wasn't sufficient."

**Problem: Competitive pricing intelligence is stale by the time it reaches sales**
Solution: Set up Visualping or Wappalyzer to monitor competitor pricing pages daily. Create a Zapier automation: pricing page change detected → Slack alert → auto-draft competitive response brief via Claude API → RevOps review → publish to Slack #competitive-intel channel. Target: sales has updated intel within 4 hours of a competitor pricing change.

**Problem: TCO model lacks credibility with CFO-level buyers**
Solution: Replace internal assumptions with third-party benchmarks (Gartner, IDC analyst cost data, peer company benchmarks from your own customer base). Add a "How we calculated this" footnote. Build in a CFO-facing sensitivity analysis: "What if your analyst saves only 5 hrs/week instead of 15?" Show the model still works at conservative assumptions. Offer to send a customized version post-call — this creates a follow-up reason to stay in contact and reinforces credibility.
