8. **Sales enablement snippet** (2–3 bullet points): Designed for SDRs to paste into outbound emails as a conversation starter.

9. **Podcast guest talking points** (5 bullets): If the author were a podcast guest, what would they say?

10. **Short-form video hook variations** (5 hooks, 1 sentence each): For LinkedIn video, YouTube Shorts, or TikTok.

For each format, produce the actual content — not descriptions of what to write. Make it specific, opinionated, and ready to post.

## Integration Tips

**HubSpot:**
- Create a custom content property called `Derivative Source Asset` and populate it for every repurposed piece; this enables you to run pipeline attribution by original asset
- Use HubSpot's Campaign tool to group all derivatives under one campaign object so revenue attribution rolls up correctly
- Set up a list in HubSpot to segment contacts who engaged with ≥3 derivatives from the same asset — these are high-intent buyers ready for sales outreach

**Notion / Airtable:**
- Build a "Content Atom" database with fields: Source Asset, Derivative Type, Channel, Publish Date, Status, Engagement Metric
- Use a `Repurposing Sprint` template: one row per source asset, child rows for each derivative with owner and due date
- Connect to a Slack workflow that notifies the team when a derivative is approved and ready to schedule

**Buffer / Sprout Social / Hootsuite:**
- Bulk-schedule the LinkedIn and X/Twitter derivatives immediately after running this prompt — don't let the content sit unpublished
- Tag every scheduled post with the source asset label so reporting can group performance by content investment

**Zapier Automation:**
- Trigger: New webinar recording added to Google Drive → Zap extracts transcript → sends to GPT-4 with this prompt → outputs derivatives to a Notion database → notifies content team in Slack
- Trigger: New blog post published in WordPress → auto-generate 3 LinkedIn post variations using the post content → send to Buffer draft queue for approval

**Salesforce:**
- Link repurposed sales enablement snippets to a Campaign Member record so you can track which content assists in pipeline stages
- Use Salesforce Content Builder to house SDR snippets with tagging by `Source Asset` and `Use Case`

## Troubleshooting

**Problem: The repurposed content feels generic and loses the original's specificity**
Solution: The quality of the Quick Copy-Paste prompt output is directly proportional to the specificity of your "3 core insights" input. Replace vague summaries ("we found that attribution matters") with precise, data-backed statements ("we found that 68% of HubSpot MQLs attributed to marketing are actually SDR-sourced"). If you don't have specific numbers, use the Advanced Version's MODULE 1 to extract them from a transcript or document first.

**Problem: LinkedIn posts all sound the same after running this prompt multiple times for different assets**
Solution: Vary the "brand voice" parameter in the Advanced Version between runs. Rotate through: (1) Data-led/analyst voice, (2) Practitioner/how-to voice, (3) Contrarian/challenger voice. Also explicitly ask the model to avoid sentence structures it used in the previous post by pasting the last post and saying "do not open with a similar structure."

**Problem: Sales team isn't using the enablement snippets**
Solution: The snippets must be in the tool the reps already use (Salesloft, Outreach, Apollo — not email). Format matters: reps skip anything over 3 sentences. Re-run MODULE 4 with the constraint "maximum 2 sentences per snippet, must end with an open-ended question, do not pitch — only provoke curiosity." Then load directly into sequence templates with the SDR manager's approval.
