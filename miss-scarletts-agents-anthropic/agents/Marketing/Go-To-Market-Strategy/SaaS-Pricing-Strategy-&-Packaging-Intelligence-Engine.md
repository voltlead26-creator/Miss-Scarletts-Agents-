## Integration Tips

- **Stripe / Chargebee:** Export the tier structure directly into product catalog configuration. Use metered billing for usage-based components. Set up automatic upgrade prompts at usage thresholds via webhook → HubSpot workflow.
- **HubSpot / Salesforce:** Create custom "Pricing Tier" deal property, set up discount approval workflows (sales manager approval required for >20% discount), track deal loss reason = "price" vs "features" separately to measure pricing effectiveness.
- **Intercom / Pendo:** Trigger in-app upgrade prompts at Kano-defined upgrade moments (e.g., 80% of project limit used). Send to Segment as "upgrade_prompt_shown" event to track conversion in your data warehouse.
- **Notion / Google Sheets:** Use the tier architecture table as your internal pricing bible. Share with Sales, CS, and Finance so everyone negotiates from the same floor.
- **Zapier:** Automate PQL alert when trial user hits all 3 activation milestones → assign to SDR in CRM → send personalized pricing email via HubSpot.

## Troubleshooting

**Problem: The AI recommends price points that feel too high or too low for your market.**
Solution: Add more competitor pricing data and customer ROI evidence to the prompt. Include 2-3 real customer testimonials with specific ROI numbers (e.g., "saved 12 hours/week"). The model calibrates to evidence — the more data you give, the more accurate the output. Also run the Van Westendorp prompt output through a quick survey with 10 existing customers to validate.

**Problem: The tier feature allocation feels arbitrary and doesn't create clear upgrade motivation.**
Solution: Add your full feature list to the prompt and explicitly label each feature as: (A) used by 80%+ of customers, (B) used by power users only, or (C) requested most in sales calls. This usage data lets the model apply Kano logic correctly rather than guessing which features to gate.

**Problem: The pricing strategy conflicts with existing customer contracts or creates grandfathering complexity.**
Solution: Add a section to your prompt: "We have [X] existing customers at [current price]. Describe a grandfathering approach that preserves trust while transitioning to new pricing over [6/12] months." The model will generate a migration plan with communication templates, grace periods, and opt-in incentives for early upgrades.
