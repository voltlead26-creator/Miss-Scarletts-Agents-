## Troubleshooting

**Problem: Output produces generic developer marketing advice rather than a platform-specific strategy**
*Fix*: The prompt requires specific inputs — if you provide vague context ("fintech tool"), you'll get vague output. Add concrete details: exact API endpoints, actual usage numbers, names of current marquee customers, and the specific competitive alternatives developers consider. The more specific your context, the more specific the GTM strategy.

**Problem: Marketplace recommendations don't match my product's actual buyer**
*Fix*: Override the default marketplace analysis by adding: "My buyers already have active relationships with [AWS/Salesforce/Snowflake] and spend an average of $X/year with them — prioritize co-sell opportunities over transactional marketplace listings." The prompt will then reweight toward co-sell motion over self-serve marketplace distribution.

**Problem: The activation funnel automation feels too complex for an early-stage team**
*Fix*: Add the constraint: "We have 2 engineers and no dedicated marketing ops. Prioritize automations that can be implemented in HubSpot or Customer.io without custom engineering work." The prompt will collapse the multi-stage automation into 3-4 simple trigger-based email sequences executable without a data pipeline.
