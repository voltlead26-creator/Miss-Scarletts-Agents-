# Customer Proof Library & Social Proof Automation Engine - Mine, Structure, and Deploy Customer Evidence at Scale

**Difficulty:** Advanced | **Time:** 20-30 min | **Tags:** b2b, content, automation, social-proof, sales-enablement, conversion

## Overview
This engine systematically mines every customer touchpoint — G2/Capterra reviews, NPS responses, Zoom call transcripts, Slack messages, CRM notes, and support tickets — to build a segmented, channel-ready proof library that sales, marketing, and CS teams can deploy instantly. Use it whenever you need to eliminate "we don't have enough social proof" as an excuse and turn your existing customer data into a conversion machine.

## Integration Tips

- **HubSpot CRM:** Create a custom "Proof Asset" property on the Contact/Company object. Tag closed-won deals with the proof assets generated from their win. Use HubSpot workflows to auto-ping CS team 90 days post-close to trigger the NPS → Proof pipeline.
- **Salesforce:** Build a Proof Library as a custom object linked to Accounts. Sales reps can filter by Vertical + Objection to pull relevant proof in 2 clicks during a call. Add Salesforce CPQ integration to auto-insert matching proof into proposal templates.
- **Notion:** Create a Proof Library database with properties: ICP Match, Vertical, Channel, Objection Addressed, Asset Type. Filter views for each team (Sales, Content, Demand Gen). Embed in sales playbook pages.
- **Zapier/Make:** Trigger → NPS response received (Delighted/Typeform) with score ≥ 8. Action → Send response text to Claude API with this prompt. Output → Auto-populate Notion database row + Slack alert to marketing team with generated assets.
- **G2/Capterra Alerts:** Set up weekly review digest → pipe new reviews into this prompt → review outputs → approve and push to Salesforce and Notion in one workflow.
- **Gong/Chorus:** Export call highlight reels where customers mention ROI or competitor comparisons → batch-process through this prompt weekly → route high-scoring outputs to content team for development.

## Troubleshooting

**Problem:** The AI generates generic, vague proof assets that feel hollow and won't influence buyers.
**Solution:** The issue is almost always thin input. Reviews like "great product, love the team" are proof-free. Before running this prompt, filter your raw evidence to only include responses of 50+ words or those that mention a specific outcome, pain point, or comparison. Quality of input determines quality of output — curate before you automate.

**Problem:** Sales reps aren't using the proof library even after it's built.
**Solution:** Proof libraries die when they're a destination, not embedded in workflow. Use the Salesforce/HubSpot integration tips to surface proof inside the tools reps already live in. Create a "Proof of the Week" Slack bot that posts one high-impact asset to #sales every Monday. Run a 15-minute proof library walkthrough in the next sales QBR showing how to pull proof in under 60 seconds during a live discovery call.

**Problem:** Customers are hesitant to be quoted publicly, so the library ends up anonymized and loses credibility.
**Solution:** Offer tiered proof levels: (1) Named quote with logo for public use, (2) Named quote with company size/industry only, (3) Full anonymization. Even heavily anonymized proof converts when the context is specific enough ("A 1,800-person logistics company cut X by Y"). Use the Proof Collection Campaign output from Step 4D to offer meaningful incentives — featured placement in a co-marketed case study drives more opt-ins than gift cards.
