**Difficulty:** Advanced | **Time:** 20 min | **Tags:** customer-success, implementation, technical-onboarding, enterprise, time-to-value, b2b-saas, project-management

## Overview
Automates the full lifecycle of enterprise SaaS implementation projects — from kickoff planning through go-live certification — by generating implementation plans, milestone trackers, risk assessments, stakeholder communications, and go-live readiness scores. Use it when onboarding enterprise accounts with complex integrations, multi-team deployments, or implementation timelines longer than 30 days.

Generate a 25-point go-live readiness assessment across 5 dimensions:

**Dimension 1: Technical Readiness (8 points)**

**Dimension 2: User Readiness (5 points)**

- Hands-on practice in UAT environment: [Target ≥ 2 sessions per user group] [Pass/Fail]
- Power user cohort trained and designated as internal help desk [Pass/Fail]
- Job aids, quick reference cards, and help center links distributed [Pass/Fail]
- Manager briefing completed (managers understand what changes Day 1) [Pass/Fail]

**Dimension 3: Business Process Readiness (5 points)**
- SOPs updated to reflect new system workflows [Pass/Fail]
- Legacy system decommission plan (or parallel-run plan) finalized [Pass/Fail]
- Reporting and dashboard requirements confirmed and built [Pass/Fail]
- Approval workflows and business rules validated by process owners [Pass/Fail]

**Dimension 4: Support Readiness (4 points)**
- Hypercare support model defined (hours, escalation path, response SLA) [Pass/Fail]
- Customer internal triage process documented (who handles Tier 1 questions) [Pass/Fail]
- Known issues log reviewed and accepted by customer [Pass/Fail]
- Post-go-live check-in schedule confirmed for first 30 days [Pass/Fail]

**Dimension 5: Executive Sign-Off (3 points)**
- Executive sponsor has reviewed readiness summary and confirmed go-live decision [Pass/Fail]

**Scoring:**
- 23-25: ✅ Go-Live Certified — proceed with confidence
- 19-22: ⚠️ Conditional Go-Live — document accepted risks, escalate to leadership
- <19: 🚫 Go-Live Blocked — escalate immediately, reset timeline

### DELIVERABLE 5: HYPERCARE & FIRST-30-DAYS SUCCESS PLAN

Generate a structured hypercare program for the first month post-go-live:

**Week 1 (Days 1-7): Stabilization**
- Daily 15-minute standups with customer PM

- Issue triage protocol: Severity 1 (system down) → 1 hr / Severity 2 (workflow broken) → 4 hrs / Severity 3 (cosmetic/non-blocking) → 48 hrs
- End-of-week health check: user login rates, core workflow completion rates, support ticket volume

**Week 2-3 (Days 8-21): Adoption Acceleration**
- Identify and engage laggard user groups (adoption below 50%)

**Week 4 (Days 22-30): Value Validation**
- 30-Day Success Review meeting with executive sponsor
- Generate first ROI data point (even if partial)
- Surface 1-2 quick wins for customer case study
- Establish steady-state cadence (transition from hypercare to standard QBR rhythm)
- CSM handoff package: implementation summary, open items, success plan for next 90 days

## Integration Tips

- **Gainsight / Totango / ChurnZero:** Map each implementation milestone to a Success Plan task in your CS platform. Use the go-live readiness score as a custom field to trigger automated health score calculations during implementation. Set up timeline entries from the weekly status update output automatically via API.
- **Salesforce / HubSpot CRM:** Log implementation milestones as Activity records linked to the Opportunity. Use the risk register flags to trigger alerts to the AE when the account shows Red status — the AE needs to know renewal risk before it's obvious.
- **Project Management Tools (Asana, Monday.com, Jira):** The Implementation Master Plan maps directly to a project template. Create one template per implementation tier (Standard / Complex / Enterprise) and auto-populate with the AI-generated plan at contract close.
- **Slack / Microsoft Teams:** Set up a dedicated #impl-[account-name] channel with the hypercare communication plan as the pinned message. Use Zapier to post automated status alerts when milestone RAG status changes to Red.
- **Notion / Confluence:** Build an Implementation Intelligence Hub with the full deliverables set. Share the Executive Sponsor 1-pager as a Notion page with comment access — executives engage more when they can comment inline vs. read a PDF.
- **DocuSign / PandaDoc:** Convert the Go-Live Readiness Certification into a formal sign-off document. Requiring the customer to digitally sign the go-live certification creates shared accountability and legal clarity if issues arise post-launch.
- **Zapier / Make Automation:** Trigger the weekly status email template generation automatically every Friday at 9am from your project management tool's milestone data → AI fills the template → routes to IM for 5-minute review → sends to customer.

## Troubleshooting

**Problem: The implementation plan is too generic — it doesn't reflect my product's actual technical complexity.**
Fix: Add a "Technical Architecture" section to your input with the specific integration patterns your product uses (REST API vs. webhook vs. ETL), your standard environment provisioning steps, and any product-specific validation requirements. The more precise your technical scope input, the more accurate the milestone dependencies and technical risk flags. Include your actual go-live definition (what must be true on Day 1 vs. what's Phase 2).

**Problem: The executive sponsor communication is too long — the CNO/CFO won't read past the first paragraph.**
Fix: Add this constraint to your prompt: "Rewrite the Executive Sponsor Update for a C-suite executive with a 90-second reading window. Max 200 words. Use a stoplight icon (🟢/🟡/🔴) as the first character so they see status before reading a word. Lead with business impact, not project activity. Cut all technical language." Brevity is respect for executives — the prompt defaults to complete but trims cleanly with this instruction.

**Problem: The risk register flags risks but the mitigations are too vague to action.**
Fix: Add specific context about your past implementation failures: "Our #1 implementation failure mode is [customer IT team deprioritizing our work during competing projects]. Our most effective mitigation has been [scheduling a 30-minute weekly sync with IT lead locked in before kickoff]." When the AI knows your specific failure patterns, it generates mitigations that match your real-world experience, not generic project management theory.
