## Integration Tips

**HubSpot:**
- Build a "Pre-Event Campaign" enrollment list using smart list criteria: ICP job title + company size + geographic proximity (for in-person) — use HubSpot's proximity filter with event zip code
- Use Sequences for Tier 1 ABM outreach (sales-owned, personalized); use Marketing Emails for warm prospect nurture
- Create a "Registered" deal stage in the events pipeline to track registration-to-pipeline conversion
- Set up registration webhook from Splash/Eventbrite → HubSpot contact property update via Zapier or native integration
- Build a registration velocity dashboard in HubSpot Reports: registrations by week vs. target, broken out by source (email, paid, organic, partner)

**Salesforce + Marketing Cloud:**
- Use Journey Builder for the warm prospect email sequence; set entry criteria as "ICP segment = true AND event registration status = not registered"
- Use Salesforce Campaigns to track each channel's registration contribution; set primary campaign source on registration record
- Build a pre-event Einstein activity score using event-related engagement signals (email opens, ad clicks, landing page visits) as a proxy for attendance likelihood

**Splash / Hopin / Bizzabo (Event Platforms):**
- Enable UTM tracking on registration links per source (utm_source=email_tier1, utm_source=linkedin_awareness, etc.) to attribute registrations to specific campaign touchpoints
- Set up automated confirmation email from event platform; suppress HubSpot/Marketo "registered" trigger email if event platform sends one (prevent double sends)
- Use event platform's waitlist feature as a scarcity signal for in-person track: "Join waitlist" copy converts well for high-demand events

**LinkedIn Campaign Manager:**
- Build a "Event Registrant Lookalike" audience once you have 100+ registrations — upload the list and let LinkedIn find similar profiles
- Use LinkedIn's "Lead Gen Form" ad format for virtual events (eliminates registration page friction; captures LinkedIn profile data automatically)
- Set up a "Registered" conversion event on your registration confirmation page and optimize LinkedIn campaigns for this conversion

**Google Analytics / GA4:**
- Set up a custom event `event_registration_complete` on form submission confirmation; track registration source in GA4 custom dimensions
- Build a pre-event registration funnel: landing page visits → form starts → form completions; identify drop-off points for CRO
- Create a GA4 audience of "registration page visitors, did not complete" and sync to Google Ads for retargeting

## Troubleshooting

**Problem: Registration pace is at 30% of goal with 3 weeks remaining**
Solution: Activate your emergency registration sprint: (1) Send an unscheduled personal email from a named executive to all Tier 1 and Tier 2 contacts who haven't registered — keep it under 100 words, acknowledge you're making one last push, be honest about the urgency; (2) Double paid media budget for the final 3 weeks, shifting entirely to retargeting and conversion campaigns; (3) Activate all sponsors and speakers simultaneously with a single co-promotion push in the same 5-day window to create a concentrated visibility spike; (4) If event is invite-only or curated, temporarily open general registration to expand the pool. Do NOT simply send more generic emails to your existing list — they've already ignored them. New channels and new messengers (speakers, sponsors, SDRs) are your levers.

**Problem: High registration volume but low ICP quality (wrong job titles, wrong company sizes registering)**
Solution: Your targeting is too broad. Diagnose by source: if LinkedIn ads are the culprit, add exclusions (company size below ICP floor, exclude student/intern titles) and shift budget from interest-based to job-title-specific targeting. If email is the issue, clean your segmentation — ensure your warm prospect list was filtered by ICP criteria before enrollment. For in-person events with capacity constraints, add a registration qualification question (e.g., "What is your primary job function?" or "Company size") and use responses to manually approve or auto-approve based on ICP match rules. Gate the confirmation email behind approval to maintain quality without rejecting at the door.

**Problem: Show rate is tracking low despite high registrations (pre-event survey or calendar acceptance rate is poor)**
Solution: Show rate correlates with registration motivation and proximity to event. For in-person: send a "what to expect" logistics guide 2 weeks out (agenda, parking, dress code, networking format) — people who receive logistics info cancel less because they've mentally committed. For virtual: low show rate is almost always a subject line problem on reminder emails — A/B test "Your session starts in 2 hours: [Speaker Name] on [Topic]" vs. generic "DataEdge Summit starts today." Also: adding a personal calendar invite from the SDR who owns the account (not from the event platform) dramatically improves show rate for Tier 1 accounts.
