## Integration Tips
- **Skilljar / Northpass / Thought Industries (dedicated LMS):** The gold standard for 50+ accounts. Configure SSO so users log in with their product credentials — removes friction and lets you correlate learning data with product usage data natively. Use the LMS API to push certification completion events to HubSpot or Salesforce as Contact properties: `certification_tier`, `certification_date`, `modules_completed`. This enables all downstream automation triggers.
- **Notion + Loom (no LMS budget):** Fully viable for up to ~100 accounts. Build a Notion Academy landing page with a structured table of contents. Embed Loom videos directly in Notion pages. Use Typeform for assessments and pass/fail scoring. Award certifications manually by sending a PDF badge and adding a contact property in HubSpot. Upgrade to a real LMS when education team headcount exceeds 1 person or when you need cohort reporting.
- **Gainsight:** Create a Timeline activity type called "Certification Earned" that auto-logs when a user passes any tier. Set up a Scorecard dimension called "Education Engagement" using the formula above. Configure a CTA type "Certification Gap" for accounts where <50% of licensed users have completed Tier 1. Add education completion to your QBR slide template as a standard data pull.
- **HubSpot:** Build a Contact property `highest_certification_tier` (None / Tier 1 / Tier 2 / Tier 3). Create a Workflow: when `highest_certification_tier` changes to "Tier 2" → create an Expansion deal in the pipeline (stage: Education-Qualified) assigned to the paired AE → send congratulations email from CSM → add a task: "Discuss expansion readiness in next QBR." This single automation drives measurable expansion pipeline from education.
- **Salesforce + Trailhead (inspiration model):** If you want to build a Salesforce Trailhead-style gamified experience, use a platform like LearnUpon or Docebo with custom branding. Map each certification tier to an Account field `[Vendor]_Certification_Level` synced via native integration. Use this field in account scoring models and in renewal risk calculations.
- **Zapier (low-code integration):** If using a basic LMS without native CRM integration, build a Zapier workflow: LMS assessment passed → create/update HubSpot Contact property `certification_tier` → send internal Slack notification to CSM → enroll contact in post-certification email sequence. This covers 90% of the automation needed without engineering support.

## Troubleshooting
- **Problem:** Customers register for the academy but drop off before completing the first module.
  **Solution:** The most common cause is unclear value framing — users don't know what they'll be able to do after completing the module that they can't do today. Rewrite every module intro to lead with the specific workplace outcome: "After this 10-minute module, you'll be able to [specific task] without asking your CSM." Drop completion data from the first 30 days and identify the exact lesson where users abandon — this pinpoints the friction point. Typical fix: the abandoned lesson is either too long (>15 min) or requires the user to complete a setup step in the product that they haven't done yet. Split long lessons into two shorter ones and add prerequisite setup instructions at the start.

- **Problem:** CSMs are being asked to deliver live training sessions even after the academy launches, undermining the self-serve education model.
  **Solution:** This happens when CSMs don't have a redirect script and when the academy doesn't visibly answer the question the customer is asking. First, give every CSM a single response template: "Great question — we actually have a 9-minute module in the Mosaic Finance Academy that walks through exactly this: [direct link to specific lesson]. Most customers find it faster than waiting for a live session. Want me to send you the link and check in after you've watched it?" Second, audit your top 10 support ticket categories and verify there is a direct link from each FAQ answer to a specific academy lesson. If a common question isn't covered by any module, it's a curriculum gap — add it to the next content sprint.

- **Problem:** Certified customers are still churning at similar rates to non-certified customers, and education isn't showing the expected retention lift.
  **Solution:** Certification alone doesn't prevent churn — application does. If customers are passing assessments but not changing behavior in the product, the content is testing recall (trivia) rather than application. Audit your assessment questions: every question should describe a real scenario and ask what the learner would do, not what a term means. Second, check your lab design — if labs are in a sandbox environment rather than the learner's live account, the behavior change doesn't transfer. Require at least one lab task to be completed in the learner's production environment, with a screenshot submission as proof. Third, tie certification to a tangible unlock (advanced feature access, dedicated Slack channel, reduced support SLA) — when the badge has no real-world benefit, completion motivation drops and the program becomes box-checking rather than genuine skill building.
