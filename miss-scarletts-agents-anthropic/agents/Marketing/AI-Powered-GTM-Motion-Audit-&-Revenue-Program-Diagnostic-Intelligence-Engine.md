## Integration Tips

- **Salesforce / HubSpot CRM:** Pull program-level pipeline influence reports before running the audit. Use Campaign Attribution reports in Salesforce or Campaign ROI reports in HubSpot. Export as CSV and paste directly into the prompt's Program Inventory section.
- **Google Analytics 4 / BI Tools:** Pull Acquisition Source → Pipeline and Revenue reports from your BI tool (Looker, Tableau, Metabase) for the last 90 days. GA4 Explorations can show Conversion Path by channel for multi-touch view.
- **Gong / Chorus:** Run a "Deal source" filter on closed-won deals to validate self-reported attribution vs. conversation-based attribution. Often reveals that event-sourced deals close at higher rates than CRM attribution shows.
- **6sense / Bombora / Demandbase:** If using intent tools, pull "account stage by channel" to see which programs are activating accounts at each buying stage. This reveals which programs are demand creation vs. demand capture.
- **Notion / Confluence:** Export audit results as a structured program portfolio page. Use the Board-Ready Executive Summary as the CMO's marketing operating system page for quarterly reviews.
- **Zapier / Make:** Automate monthly program performance data collection by building a flow: CRM → Google Sheets → monthly performance summary → prompt re-run for ongoing audit cadence.

## Troubleshooting

**Problem: My attribution data is unreliable — programs are either over- or under-credited.**
Solution: Run a parallel self-reported attribution analysis. Ask recent closed-won customers in onboarding or QBR calls: "What marketing touchpoints do you recall before you reached out to us?" Weight self-reported data at 30% alongside CRM attribution data. Also use revenue influence (multi-touch) rather than sourced-only attribution to give programs partial credit.

**Problem: The AI recommends killing a program that an executive champion is attached to (e.g., CMO's pet project or CEO's preferred conference).**
Solution: Re-run the audit section for that program with explicit constraints: "Assume this program is non-negotiable. How do I optimize it to get closer to the portfolio efficiency target while reallocating the remaining freed budget?" This creates a defensible optimization plan that respects political realities without compromising the overall rationalization.

**Problem: My company is early-stage with limited historical data — I don't have 90 days of program-level attribution.**
Solution: Use the Quick Copy-Paste version and add: "I have limited historical data — use industry benchmark ranges for B2B SaaS [stage] to estimate program efficiency where actuals are unavailable, and flag which recommendations require data validation before execution." Run the full audit again in 60 days once you have baseline data.
