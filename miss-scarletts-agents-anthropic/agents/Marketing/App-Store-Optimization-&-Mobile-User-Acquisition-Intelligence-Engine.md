## Troubleshooting

**Problem: Keyword rankings not improving after metadata update**
- Fix: Apple App Store requires 2-4 weeks for full index update — track weekly. Verify keywords are not in title AND keyword field simultaneously (wastes character allocation). Use a tool like AppFollow or Sensor Tower to confirm indexing. Check if your app has enough reviews/ratings velocity to rank competitively for mid-tier keywords.

**Problem: CVR dropped after updating screenshots**
- Fix: Revert to previous screenshots immediately if available via Custom Product Pages. Run a proper A/B test through Product Page Optimization (Apple) or Google Play Experiments before going live. Common mistake: testing too many variables at once. Test title OR screenshots, never both simultaneously.

**Problem: App Store review prompt not triggering enough reviews**
- Fix: Ensure you're using the native `SKStoreReviewRequest` API (Apple) or In-App Review API (Google) — custom prompts that redirect to the store violate guidelines and can trigger de-listing. Check that you haven't exceeded the 3-times-per-365-days limit Apple enforces. Move trigger earlier in user journey — most apps prompt too late. Consider segmenting: only prompt users who complete a core value action (e.g., successfully submit first expense report).
