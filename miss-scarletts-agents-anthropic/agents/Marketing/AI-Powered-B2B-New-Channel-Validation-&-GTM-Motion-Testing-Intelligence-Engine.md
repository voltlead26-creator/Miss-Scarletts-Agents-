SECTION 6: FAILURE MODE ANALYSIS

Anticipate and build detection for these common validation failures:

FALSE NEGATIVE (killing a good channel prematurely):
- Cause 1: "Wrong message, not wrong channel" — Test detected poor performance but the audience is correct; the creative or offer didn't resonate
  * Detection: High engagement rate but low conversion rate (indicates curiosity without intent match)
  * Correction: Run 1 creative pivot test before killing
- Cause 2: "Insufficient ramp time" — B2B buying cycles mean pipeline doesn't appear until week 10+
  * Detection: Strong Tier 1-2 indicators but no Tier 3 pipeline yet
  * Correction: Extend validation timeline for high-ACV, long-cycle deals; use opportunity-stage indicators rather than closed-won
- Cause 3: "Attribution failure" — Channel is working but attribution model misses credit
  * Detection: Branded search volume or direct traffic increases during channel run periods
  * Correction: Run MMM correlation analysis; survey recent customers on where they first heard of you

FALSE POSITIVE (scaling a channel that won't sustain ROI):
- Cause 1: "Novelty effect" — Early cohorts convert at high rates because they're the most interested segment; performance degrades as you reach lower-intent audiences
  * Detection: Week-over-week CPL trends upward as volume scales
  * Correction: Monitor CPL trajectory on a rolling 2-week basis; model saturation curve before committing to full scale
- Cause 2: "Double-counting" — Leads counted from this channel were also generated by existing channels running concurrently
  * Detection: High overlap between channel-attributed leads and email-nurtured or paid-search-exposed accounts
  * Correction: Apply de-duplication rules and incrementality controls
- Cause 3: "Regression to mean" — Test launched during unusually high demand periods (product launch, competitor outage, industry news)
  * Detection: Compare test period demand metrics to rolling 90-day baseline
  * Correction: Wait for demand normalization; re-run test in a neutral period

## Integration Tips

**HubSpot:** Create a custom "Channel Validation" deal pipeline stage. Tag all opportunities with `channel_test_cohort` property. Build a report showing CAC and pipeline by validation cohort to compare test vs. control performance in real time.

**Salesforce:** Add a Campaign Influence model specifically for new channel tests — use "First Touch" attribution for validation tests (not multi-touch) to isolate the new channel's contribution without dilution from existing nurture.

**Google Analytics 4:** Create a dedicated GA4 Exploration for each channel validation test. Set up custom events for channel-specific conversion actions (promo code use, UTM landing page visit, form completion with source attribution). Build a funnel visualization showing drop-off at each stage.

**Looker/Tableau:** Build a "Channel Validation Dashboard" with toggle for test vs. control period, weekly CAC trend line, leading indicator gauges with red/yellow/green thresholds, and Bayesian posterior probability calculator updating in real time.

**Zapier/Make:** Automate the weekly channel validation report: pull CRM data → calculate CAC and pipeline metrics → generate Bayesian posterior probability → send Slack alert if any metric hits a kill threshold. This ensures no decision gate is missed due to reporting lag.

**6sense/Bombora:** For awareness-oriented channels (podcast, display, CTV), use intent data as a Tier 2 proxy for pipeline. Track intent score increases for accounts in the ICP firmographic profile during the test window — a statistically significant lift vs. control indicates demand creation even before form completions appear.

## Troubleshooting

**Problem: Can't isolate channel effect — too many campaigns running simultaneously.**
Fix: Run a holdout group using geographic or firmographic segmentation (e.g., test channel only in West Coast accounts, hold East Coast as control). This requires CRM suppression lists and coordination with SDR team to not call control-group accounts during the test period. Even imperfect holdouts are better than no comparison group.

**Problem: Test period is too short to see pipeline from long-cycle deals (90-180 day cycles).**
Fix: Shift primary success criterion from "opportunities created" to "Tier 2 leading indicators" (account-level intent lift, form fill attribution, MQL rate). Extend the tracking window to 16 weeks post-campaign launch and set calendar reminders for delayed pipeline review. For enterprise ACV deals, a 12-week validation window is often insufficient — build a "deferred validation" process where you revisit channel economics at 6 months.

**Problem: Team wants to scale before hitting posterior probability threshold because the channel "feels like it's working."**
Fix: Enforce the Bayesian decision gate process by presenting posterior probability as a literal number at each review meeting. Pre-agree on the threshold (e.g., 80% confidence) before launching the test — not after seeing early results. Anchoring the team to pre-specified criteria prevents confirmation bias from overriding the data.
