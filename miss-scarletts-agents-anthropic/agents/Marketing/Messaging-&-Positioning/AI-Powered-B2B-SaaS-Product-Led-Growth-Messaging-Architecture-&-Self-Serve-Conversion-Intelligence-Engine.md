### MODULE 1: ACQUISITION MESSAGING ARCHITECTURE

**1.1 Homepage & Acquisition Copy System**
Write these copy elements optimized for organic, self-serve discovery (not for enterprise procurement):

- **Hero Headline Options (3 variants):** Test for clarity, outcome, and emotion. Each under 10 words. Example frameworks: [Product] + [for whom] + [outcome]; Problem statement → implied solution; Bold category claim.
- **Hero Subheadline:** 1-2 sentences expanding the headline with specificity. Name the primary user persona, the functional job they're getting done, and one measurable outcome.
- **Primary CTA Copy:** For the free tier button. Never "Get Started" — write 3 options that communicate instant value access.
- **Social Proof Hook:** One-line trust signal optimized for end-user credibility (not enterprise logos). Reference community size, developer ecosystem, or peer usage.
- **Feature Value Props (3-4):** Each one: Icon + 4-word label + 1-sentence explanation written in second person, present tense, active voice. What does the user *do* with this feature, not what it *is*.

**1.2 Signup Friction Elimination Messaging**
For each step of the signup flow, write copy that maintains momentum:

- **Email capture page headline:** What do they get immediately after signup? Not features — outcomes.
- **Email verification interstitial:** What happens while they wait? Bridge this moment with a promise statement.
- **Profile/onboarding question framing:** If you ask setup questions, write them as value-delivery questions: "We'll customize your [product noun] based on this" not "We need your information."
- **Progressive disclosure rationale:** When asking for company size, role, or use case — explain the user-facing value ("so we can show you relevant [templates/integrations/examples]").

**1.3 Free Tier Positioning**
Write the narrative that defines what free means without cheapening the product:

- The "what you get free forever" statement (generous framing, not limitation framing)
- The "upgrade when you're ready" value anchor (what specifically changes after upgrade)
- The "no credit card required" trust signal copy (say it differently — everyone says this)

### MODULE 2: ACTIVATION MESSAGING SYSTEM

Apply the Hook Model (Cue → Action → Variable Reward → Investment) to each activation stage.

**Stage 0 — Zero State (First Login, No Data)**
The product is empty. This is the most dangerous moment for churn.

- **Empty state headline:** Don't say "No [items] yet." Write a headline that frames the empty state as an exciting starting point. Include what's possible, not what's missing.
- **Primary action CTA:** The single action users must take first. Write the button copy and the 1-sentence supporting rationale beneath it.
- **Quick win guide copy:** A 3-step "Start in 5 minutes" guide using time-boxed language. Each step = action verb + outcome. No more than 12 words per step.

**Stage 1 — First Action (Milestone M1)**
User takes the first meaningful product action (connect a source, import data, create first item).

- **Progress indicator copy:** Write the progress bar label and milestone name for M1. Use language that rewards action, not just records it.
- **Triggered email: "You just did [M1]" (send immediately):** Subject line + 3-sentence body. Tone: celebratory but not patronizing. Include one next-step CTA with the specific next milestone named.
- **In-product success toast:** 1 line. Acknowledge the action, preview the next step.

**Stage 2 — Aha Moment (Primary Value Delivery)**
User experiences the core product value for the first time.

- **In-product Aha acknowledgment:** An in-product banner or modal that names what just happened in terms of user outcome (not product feature). Write: Headline (what they achieved) + 1-sentence implication (what this means for them) + CTA (save, share, or next action).
- **Triggered email: "Your first [outcome] is ready" (send within 1 hour):** Subject must reference the specific outcome, not the product. Body: 2-3 sentences narrating what happened, why it matters to their workflow, and one forward-looking CTA.
- **Share/export prompt:** If the output is shareable, write the in-product prompt that encourages virality: a question that makes sharing feel like collaboration, not promotion.

**Stage 3 — Habit Formation (Day 7–14)**
User has returned 3+ times. This is where retention messaging shifts.

- **Weekly "Your [Product] Summary" email:** Subject, opening line, and the 3 metrics/outcomes to surface. Frame metrics as "what you accomplished" not "how you used the product."
- **Streak or milestone acknowledgment:** Write a 2-sentence in-product notification for the user's 10th action, 5th session, or team first-use. Use social identity language: "People who do X tend to Y."
- **Habit anchor prompt:** A strategically placed prompt (day 7–10) that encourages users to set a recurring workflow or schedule — turning episodic use into habitual use.

### MODULE 3: PQL TRIGGER MESSAGING SYSTEM

For each PQL signal, define the exact moment, the upgrade offer copy, and the follow-through sequence.

**Signal 1: Feature Gate Hit**
User attempts to use a feature that requires upgrade.

- **In-app gate moment modal:** Write: (a) Headline that acknowledges what they were trying to do (not "Upgrade required"), (b) 2-sentence value prop for the paid tier using THEIR current usage context, (c) Primary CTA: upgrade action, (d) Secondary CTA: "See all paid features."
- **Follow-up email (sent 30 min later if no upgrade):** Subject line + 3-sentence body re-surfacing the value without feeling pushy. Include a use case example from someone like them.

**Signal 2: Usage Threshold Hit**
User reaches 80% of free tier limit (e.g., 4/5 data sources connected).

- **In-product threshold warning:** Not a "you're almost out" warning — write it as a "you're getting serious about [use case]" progress message. Headline + 1-line usage display + upgrade CTA with specific next-tier capacity named.
- **Threshold email (send when 80% limit reached):** Subject that quantifies their progress. Body: 2-3 sentences showing their usage trend + projection of when they'll hit the limit + upgrade CTA that names the specific unlock.

**Signal 3: Collaboration Signal**
User invites a teammate or shares content externally.

- **Post-invite upgrade modal:** This is a high-intent moment. Write: Headline ("Your team is joining [Product]") + social proof element (reference how many teams of similar size use paid tier) + upgrade offer emphasizing team-specific features + CTA.
- **Email to inviter (send 24 hours after invite accepted):** Subject celebrating the team milestone + body that shows what collaboration features they're missing on free tier + upgrade CTA framed as "unlock team features."

**Signal 4: High-Frequency Power User**
User is in product 5+ days/week, clearly relies on it.

- **Power user recognition modal (Day 14–21 if criteria met):** Not a sales pitch — a recognition moment. Headline: acknowledge their expertise level with product. Body: 1 sentence recognizing their sophistication + 1 sentence on what power users typically unlock at paid tier + soft CTA ("When you're ready to scale...").
- **Personalized sales touch trigger:** This is where the product-led sales handoff happens. Write the email template a sales rep (or AI SDR) sends to a power user who hasn't converted: 4 sentences max, references their specific usage data, no generic pitch, includes a calendar link with a low-friction label ("15 min — I'll show you what similar teams unlock").

**Signal 5: API / Integration Exploration**
User explores API documentation, visits integration pages 3+ times, or starts an API call.

- **Developer handoff messaging:** This buyer needs different treatment. Write: In-product message for API page visit ("Looks like you're building something interesting...") + CTA to enterprise/API plan page + short form for use case capture.
- **Developer outreach email:** Technical tone, no marketing speak. Subject: A specific technical question or use case reference. Body: acknowledge what they were trying to build, offer 1-2 technical resources, invite a brief technical conversation.

### MODULE 4: VIRAL LOOP & TEAM EXPANSION MESSAGING

**4.1 Invitation Mechanics**
Write copy for each moment in the collaboration invite flow:

- **Invite prompt (in-product):** What triggers the invite prompt? Write the contextual headline — it should feel like sharing capability, not recruiting colleagues.
- **Invite email (sent to invitee):** Subject line must name the inviting colleague and the product outcome — not just "You've been invited to [Product]." Body: 3 sentences from the product voice (as if the product is making the introduction), highlighting what the invitee can do, ending with a single CTA to the workspace.
- **Invitee landing page:** Personalized to reference who invited them. Headline: outcome-focused, references the colleague's name contextually. CTA: one action to join and see the colleague's work immediately.

**4.2 Team Growth Recognition**
When a second, third, or fifth teammate joins the workspace, send a moment-of-celebration message:

- **Team milestone email (to workspace owner, on teammate 3 and 5 joining):** Subject: Reference the team size milestone. Body: Celebrate the momentum, surface a team-specific feature they haven't used yet, include the upgrade CTA that makes sense for their now-larger team size.
- **In-product team dashboard teaser (free tier):** On the team overview page, show what a full team dashboard looks like — but gate it. Write the teaser copy that creates desire for the paid team view without frustrating free users.

**4.3 Network Effect Amplification**
PLG products gain value as more users join the same organization.

- **Network value statement:** Write 1-2 sentences that appear in the product when a user's organization already has 3+ users: "Your [colleagues] are already using [Product]. Here's what they've built..." This is the internal social proof moment.
- **Peer benchmarking prompt:** Write a contextual prompt that shows a user how their usage compares to teammates ("[Colleague] has set up 12 automations — you have 3. Here's a template to catch up."). Write this as motivational, not judgmental.

### MODULE 5: BOTTOM-UP ENTERPRISE EXPANSION MESSAGING

When individual champions are power users at organizations with expansion potential (typically >200 employees), transition to company-wide rollout language.

**5.1 Champion Identification & Activation**
The champion is the internal advocate who will sell the product to their organization. Help them with language.

- **Champion recognition email (trigger: >50 actions, invited ≥3 teammates):** Subject acknowledging their expert status. Body: Name what they've accomplished with data ("You've automated X workflows, saved approximately Y hours"), introduce the concept of company-wide rollout, offer a resource or conversation to help them make the case internally.
- **Internal selling resource offer:** Offer the champion a shareable asset. Write the CTA: "Share [Product] with your leadership team" + the 1-paragraph executive summary the champion can copy-paste into Slack or email.

**5.2 Executive Introduction Sequence**
Once a champion signals interest in broader rollout, shift to dual-thread messaging.

- **Executive summary email (for champion to forward):** Write a 200-word email the champion sends to their VP/Director. Subject + body structured as: 1 sentence on what they've been using → 2 sentences on business outcomes with specific numbers they can fill in → 1 sentence on what company-wide adoption would unlock → 1 CTA (a 20-minute briefing, not a demo).
- **Vendor evaluation summary (for procurement/IT):** 1-page messaging framework (5 sections): What [Product] does, Who in your org uses it today, Security & compliance posture (1-2 sentences), Integration compatibility with current stack, Pricing transparency. Write the section headers and opening sentences for each.

**5.3 Enterprise Upgrade Moment**
When the champion or economic buyer signals intent to formalize the purchase.

- **Proposal email from sales/CS (triggered by PQL score + account size):** Write a 4-sentence email from an AE or CSM that references their specific usage data, names 2 enterprise-tier features relevant to their use case (SSO/SAML, audit logs, dedicated support, admin controls), and includes a next step that isn't "schedule a demo" (write a better CTA).
- **Enterprise onboarding welcome email:** After deal closes, write the onboarding welcome from the CS team that connects to the PLG journey: acknowledges how they came up through self-serve, sets expectations for the enterprise experience, introduces the CSM, and sets a 30-day success milestone.

## Integration Tips

**Intercom / Pendo / Appcues (In-Product Messaging):**
- Map each Module 2 activation message to a behavioral trigger in your messaging tool. Use Pendo/Amplitude events (pipeline_run_completed, source_connected, teammate_invited) as triggers — not time-based.
- Set up "pause rules" so users who have already converted to paid don't receive free-tier upgrade messages.
- Use Intercom's series feature to build the PQL trigger sequences in Module 3 as automated journeys with manual sales rep handoff at the highest-intent signals.

**HubSpot / Marketo (Email Automation):**
- Sync behavioral PQL signals from your product analytics tool (Mixpanel, Amplitude, Posthog) to HubSpot via a Segment connection.
- Create a PLG contact lifecycle stage separate from traditional MQL/SQL: Free User → Activated → PQL → PQL-Sales Qualified → Customer.
- Build workflow branches for each PQL signal (Module 3) with a 3-day no-response branch that escalates to a sales rep notification.

**Salesforce (Enterprise Expansion):**
- When a free account's PQL score exceeds your threshold AND company size is >200 employees, auto-create an Opportunity in Salesforce assigned to the named account owner.
- The Module 5 champion recognition email is most effective when triggered automatically from Salesforce workflows based on product usage data synced from your warehouse via Reverse ETL (Census, Hightouch).

**Stripe / Billing (Upgrade Moments):**
- For the feature gate modal (Signal 1), use Stripe's checkout.session.completed webhook to trigger the post-upgrade onboarding welcome email within 60 seconds of payment.
- Surface upgrade pricing in-context (Module 3 modals) using Stripe's pricing API — avoid hardcoding prices that go stale.

**Notion / Confluence (Internal Enablement):**
- Maintain the Module 6 voice guidelines as a living document in your company wiki. Every new marketer, PM, or growth engineer working on PLG copy should read it first.
- Create a "PLG Copy Review" Slack channel where proposed copy variants get scored against the voice guidelines before shipping.

## Troubleshooting

**Problem: High signup rates, low AHA activation (common symptom: <20% of signups reach the primary AHA milestone)**

Root cause: The acquisition messaging is attracting users whose job-to-be-done doesn't match what the product actually does well. Fix: Audit your homepage hero copy for specificity — vague benefit language ("work smarter") attracts low-intent signups. Use the JTBD framework to rewrite to a specific functional job ("automate your [specific] data workflow without engineering support"). Also check: is the path from signup to first action too long? Apply Module 2, Stage 0 (Zero State) messaging to reduce friction between "account created" and "first meaningful action."

**Problem: Good PQL conversion signals, but low revenue — PQLs aren't upgrading despite reaching behavioral thresholds**

Root cause: Either your feature gate design doesn't create enough FOMO at the right moment, OR your pricing page messaging doesn't close the perceived value gap between free and paid. Fix: Audit every feature gate modal (Module 3) for the "value-before-ask" principle: does the modal make the user feel like they earned the right to an upgrade offer? If it leads with price before value, rewrite to lead with their specific usage context first. Then apply a 3-day pricing page message sequence (not just a single CTA) for users who hit the gate but don't upgrade.

**Problem: Strong individual user retention, poor team expansion — accounts stay single-user for >90 days**

Root cause: The invite flow isn't contextually triggered at the right moment, or the invitee experience doesn't immediately demonstrate value. Fix: Move your invite prompt trigger from "Day 5" (time-based) to "third pipeline run" (behavior-based). Users who invite teammates after experiencing a concrete product win have 3× higher invite acceptance rates than users prompted on a schedule. Apply Module 4.1 invite copy with the full contextual trigger design. Also: audit the invitee email (Module 4.1) — if it leads with product features rather than "your colleague [Name] built this and wants you to see it," invitees have low motivation to engage.
