## Integration Tips

**HubSpot (Content Hub + CRM):**
- Enable HubSpot's native content attribution reports under Reports → Marketing → Traffic Analytics → Content Details
- Use the "Content Interactions" property on Contact and Deal records to track which assets influenced each deal
- Build a custom deal report: filter by "Associated Content" to get pipeline influenced per asset
- Export the HubSpot Blog Analytics CSV and paste directly into the Quick prompt above

**Salesforce + Salesforce CMS:**
- Create a custom Campaign Member record for each content download with Opportunity association
- Use Campaign Influence (Primary Campaign Source model or Customizable Campaign Influence) to calculate pipeline attribution
- Build a Salesforce report: Campaigns with Influenced Opportunity Amount, filtered by Campaign Type = "Content"
- Add a Content Usage field on the Activity object so reps can log which content they share in meetings and emails

**Google Analytics 4 (GA4):**
- Set up GA4 Events for content conversions: content_download, demo_request, trial_signup with content_title and content_type parameters
- Use GA4's Path Exploration report to visualize content consumption sequences before conversion
- Build a GA4 Looker Studio dashboard with: page path, sessions, conversion rate, goal completions, revenue (if e-commerce enabled)
- Export GA4 landing page report for the relevant date range and merge with CRM pipeline data in a Google Sheet

**Looker Studio / Tableau / Power BI:**
- Build a Content Revenue Attribution Dashboard with 4 panels: STAR/WORKHORSE/AWARENESS/ZOMBIE quadrant chart, pipeline by content type bar chart, editorial velocity trend line, and sales utilization heat map
- Connect data sources: GA4 (traffic), HubSpot or Salesforce (pipeline/closed-won), content production tracker (Notion/Airtable/spreadsheet)
- Set up automated weekly data refresh so the dashboard always reflects the rolling 90-day window

**Notion / Airtable (Editorial Operations):**
- Maintain a Content Catalog database with fields: Asset Name, Type, Publish Date, Production Cost, Owner, Status, Revenue Tier (auto-updated monthly), Sales Usage Count (pulled from CRM), Pipeline Influenced (pulled from attribution model)
- Use Airtable automations to flag assets when their Revenue Tier drops to ZOMBIE (triggering a review)
- Build a monthly content review template in Notion that pulls the latest Attribution Report and prompts the content team to take action on each ZOMBIE and WORKHORSE asset

**Highspot / Seismic / Showpad (Sales Content Enablement):**
- Use native analytics to pull Sales Usage Count for each asset (views, shares, email opens from shared links)
- Export the "Content Effectiveness" report and paste into the Advanced prompt's EDITORIAL OPERATIONS DATA section
- Map sales usage data to CRM deals to calculate win rate differential: deals where content was used vs. not used

## Troubleshooting

**Problem: Sales reps aren't logging content usage in CRM, so my sales utilization data is near zero.**
Solution: Switch to tracking behavioral signals instead of manual logging. Use tools like Highspot, Seismic, or Showpad which auto-log when a rep shares a document and when a prospect opens it — this data syncs automatically to CRM. For a quick fix without new tools, check email tracking data in HubSpot or Salesloft/Outreach to see which content URLs were shared in sequences. You can also look at CRM "Campaign" records associated with deals as a proxy.

**Problem: My attribution model shows every deal was influenced by a blog post, which inflates the numbers and makes CFO/CEO skeptical.**
Solution: Apply a minimum engagement threshold for attribution credit — for example, content must have been engaged with for ≥2 minutes (time-on-page) OR triggered a conversion event to be counted as an "influence touch." In HubSpot, use "First Conversion" and "Last Interaction" reports rather than linear touch to reduce noise. Additionally, use the WORKHORSE quadrant to highlight assets where sales usage is high — that's unambiguous, behavioral evidence that content is affecting deal outcomes even without CRM attribution.

**Problem: I have hundreds of content assets and it's impossible to manually calculate pipeline attribution for each one.**
Solution: Start with your top 20 assets by traffic volume and manually map them to pipeline using your CRM's campaign influence report — this 20% of assets likely accounts for 80% of your pipeline impact. Use the ZOMBIE test as a quick filter: any asset with <50 organic sessions/month AND 0 CRM-linked conversions in 90 days can be categorized as ZOMBIE without detailed analysis. Build a Google Sheet with VLOOKUP connecting your content URL list from GA4 to your CRM pipeline data — this can be set up in 2-3 hours and automated with a monthly data refresh.
