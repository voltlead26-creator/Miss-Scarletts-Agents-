## Integration Tips

- **Google Search Console → Sheets → Claude:** Export cluster-level query data from GSC, organize by URL prefix (e.g., /blog/scheduling/ = Shift Scheduling cluster), paste into this prompt for automated cluster scoring. Use the GSC API + Google Sheets + a scheduled Claude agent for weekly automated cluster health reports.
- **HubSpot/Salesforce:** Pull "Original Source Drill-Down 1" (HubSpot) or "Lead Source Detail" (Salesforce) filtered by organic; group by landing page URL to approximate cluster pipeline attribution. Feed this data into the Advanced prompt monthly.
- **Semrush / Ahrefs:** Export keyword rankings by URL; group by cluster using subfolder or tagging logic; feed keyword visibility data alongside pipeline data for combined authority + revenue scoring.
- **Notion Content Database:** Build a Cluster Performance Dashboard in Notion that auto-updates monthly by connecting your analytics via Zapier → Notion API. Use the prompt output to populate a rolling 90-day action tracker for your content team.
- **Clearbit/6Sense:** Overlay firmographic data on organic visitors to identify which clusters attract ICP companies vs. non-ICP traffic — critical for correctly scoring pipeline influence when form-fill attribution is incomplete.
- **Automated Agent Workflow:** Set up a monthly AI agent using Claude + Python that pulls GSC data, GA4 data, and CRM pipeline data via APIs, runs this prompt, and emails the CMO a cluster performance digest every first Monday of the month.

## Troubleshooting

- **Problem:** Pipeline attribution by cluster is unavailable — CRM doesn't track organic landing page.
  **Solution:** Use a proxy metric: set up UTM-tagged organic content downloads (gated assets) per cluster and track those in CRM as a leading indicator. Alternatively, use first-touch attribution from GA4's landing page report filtered by organic source, matched against deals closed within 180 days.

- **Problem:** Results show most clusters as "Traffic Traps" — high traffic, no pipeline.
  **Solution:** This is almost always a symptom of a missing conversion path problem, not a content quality problem. Check: Does the cluster have any bottom-funnel pages (demo, pricing, ROI calculator, competitor comparison)? Is there a relevant CTA on every page in the cluster? Run a heatmap analysis (Hotjar/Microsoft Clarity) on your top organic landing pages to see where users drop off.

- **Problem:** The AI gives vague recommendations without specific page-level actions.
  **Solution:** Provide more granular input data. Instead of summarizing clusters, paste your actual top 10-20 organic landing pages with sessions + conversions for each. The more specific the input data, the more specific and actionable the output. Add: "For every recommendation, specify the exact URL, the exact change to make, and the expected CVR or traffic impact."
