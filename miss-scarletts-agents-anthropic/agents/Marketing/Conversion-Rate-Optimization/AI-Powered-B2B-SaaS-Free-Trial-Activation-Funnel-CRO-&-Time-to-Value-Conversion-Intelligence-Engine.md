## Troubleshooting

**Problem: My activation event data is incomplete — I don't have reliable in-product analytics.**
Fix: Start with what you have. Provide proxy data (Day 1 return visits, email click-through to specific features, support ticket topics) in the ENGAGEMENT LAYER section. The AI will flag which hypotheses are low-confidence and identify what data to instrument first. Use the output's measurement system section to define what to track going forward before running tests.

**Problem: The output recommendations require engineering work I can't resource.**
Fix: In the prompt, add the constraint: "All recommendations must be implementable without engineering changes, using only [list your martech tools]." This forces the AI to generate only no-code solutions (email copy, in-app guide configuration, landing page copy changes). Label any remaining engineering-dependent items as "Phase 2."

**Problem: Trial-to-paid conversion rate varies dramatically by acquisition source and I don't know which to optimize first.**
Fix: Paste source-level conversion breakdown into the prompt and add: "Prioritize recommendations for [highest-volume source] first, then provide a secondary analysis for [second source]." The segmentation model section will produce source-specific treatment recommendations that you can implement in parallel.
