## Overview
Builds a complete developer-led demand generation system that generates pipeline by earning trust in technical communities before asking for anything — through open source, technical content, DevRel programs, and community presence. Use when your ICP includes developers, data engineers, ML practitioners, DevOps/platform engineers, or technical founders who research tools on GitHub, Hacker News, Reddit dev forums, and Discord before a human ever talks to sales.

## Integration Tips

- **GitHub + HubSpot:** Use the GitHub API to pull organization domain data from contributor email addresses, enrich with Clearbit, and create company records in HubSpot. Set lifecycle stage to "Marketing Qualified" when company exceeds usage thresholds.
- **PostHog/Amplitude → CRM:** Instrument the open source product with anonymous telemetry (opt-in), aggregate by company domain using reverse IP, and trigger CRM alerts when enterprise-scale usage detected.
- **Slack/Discord monitoring → Zapier:** Use keywords alerts (your product name + competitor names) in Zapier connected to Slack channels where DevRel monitors daily. Auto-create tasks in Asana when competitor mentions spike.
- **UTM → Salesforce Campaigns:** Create a Salesforce campaign for each major developer channel (GitHub, HN, dbt Slack, etc.) and map UTM parameters to campaign membership on lead creation. Report monthly on pipeline influenced per developer channel.
- **Notion content calendar:** Build a developer content calendar in Notion with properties: Technical Depth (L1-L4), Target Community, Primary KW, CTA Type, Distribution Checklist (HN / Reddit / Twitter / LinkedIn / Newsletter). Use Notion AI to generate first drafts from brief.
- **Common Room or Orbit:** Use a community intelligence platform to track developer engagement across GitHub, Discord, Slack, Twitter, and Reddit in a unified view. Score members by engagement and flag high-value prospects for DevRel outreach.

## Troubleshooting

**Problem: Developers engage with content but don't convert to free trial.**
Solution: The friction between content and trial is too high. Audit the path: does the CTA require email signup, credit card, or enterprise contact form? Replace with "Run in 60 seconds with one Docker command" or GitHub Codespaces instant demo. Remove all gates before value delivery.

**Problem: DevRel is generating community metrics (Discord members, GitHub stars) but no pipeline.**
Solution: The DevRel OKRs are misaligned — they're optimizing for community size, not pipeline influence. Re-anchor the program: (1) require DevRel to tag every outbound community interaction with a Salesforce contact, (2) implement UTM tracking on all content, (3) build a champion pipeline report showing DevRel-sourced opportunities. If DevRel can't name 5 specific deals they influenced last quarter, the program needs restructuring.

**Problem: Internal stakeholders (sales, CEO) don't believe DLG is generating real pipeline — they want to know the ROI.**
Solution: Build a "Developer Influence Report" for the monthly revenue review: (1) show the % of closed-won deals that had a developer content touchpoint or community interaction in the journey, (2) show champion-to-opportunity conversion data, (3) run a 90-day cohort analysis comparing close rates for deals with vs. without a developer champion. Frame it as: "Deals with an active developer champion close at 2.4x the rate and at 18% higher ACV."
