Use this when a competitor makes a major announcement and you need to immediately arm sales, align messaging, communicate to at-risk customers, and turn the moment into an offensive pipeline play rather than a defensive scramble.

## Integration Tips

**Salesforce:** Create a "Competitive Move" custom object triggered when a CI alert comes in. Auto-populate related Opportunities where the competitor appears in the "Primary Competitor" field. Use Flow to alert AEs on those opportunities with the battlecard update link.

**Slack:** Build a #competitive-intel Slack channel. Post the Threat Assessment and Sales Cheat Sheet as a pinned message within 4 hours of any DEFCON 1/2 trigger. Use Zapier or native Slack API to push updates from your monitoring stack (Klue, Crayon, or manual RSS).

**Notion/Confluence:** Maintain a "Competitive Response Log" database with entries for each activation. Tag by competitor, move type, and response effectiveness score. This becomes your CI playbook library over time.

**Klue / Crayon / Kompyte:** These CI platforms have native "Battlecard Update" workflows. Feed this prompt's Phase 3 output directly into a battlecard update request; most platforms support Zapier or webhook triggers for auto-update queuing.

**HubSpot:** Use the Competitor field on deals. When a competitor is flagged, trigger a workflow that sends the sales rep the 5-bullet cheat sheet via HubSpot task notification and adds the deal to a "Competitive Deals" list for CMO/VP pipeline review.

**Gong:** After the response package is distributed, run a Gong search for the competitor's name in call transcripts over the following 2 weeks. Review whether reps are using the new talking points vs. old messaging — this is the fastest feedback loop for CI program effectiveness.

## Troubleshooting

**Problem: The announcement is vague (e.g., "we're investing in AI") with no concrete product details.**
Fix: Focus the prompt on intent analysis (Phase 1b) rather than feature response. Ask the AI to analyze job postings, recent reviews, and LinkedIn activity to infer what "investing in AI" likely means for their roadmap. Build a "watch list" battlecard section rather than a full feature response. Set a 30-day re-evaluation trigger.

**Problem: Sales team ignores the battlecard update and continues using outdated messaging.**
Fix: Change the distribution channel. A PDF or wiki page update gets ignored. Push Phase 3 (the Cheat Sheet) as a 5-bullet Slack message tagged to AEs directly in the #deals or #sales channel. Run a 15-minute optional Zoom huddle within 48 hours of the announcement — record it and add to onboarding for new reps. Make the competitive response package mandatory reading in Salesforce as an activity before any competitive deal can advance.

**Problem: The AI response feels generic and doesn't reflect our actual competitive dynamics.**
Fix: The Advanced Version requires specific inputs — if the win/loss data, ICP details, and current battlecard state are left vague, the output will be generic. Feed in 3-5 real customer quotes from recent win/loss calls, paste your existing battlecard against this competitor, and include the exact wording prospects use when they mention this competitor. The more verbatim market language you provide, the more precise and differentiated the output.
