## Success Metrics

- **Message-to-pipeline conversion rate by variant:** Primary indicator — does the message convert to qualified pipeline more efficiently?
- **Time to statistical significance:** Are experiments running long enough to be valid? Target: 95% CI within 4-6 weeks for high-traffic channels
- **Cross-channel message consistency score:** Is the winning message consistently outperforming across 3+ channels? Fragmented signals indicate ICP segmentation issue
- **Winning message propagation speed:** How quickly after declaring a winner does the entire messaging system get updated? Target: <2 weeks full propagation
- **Revenue per message variant:** Closed-won ACV attributed to prospects exposed to each variant — the ultimate arbiter
- **Experiment velocity:** Number of completed, statistically valid experiments per quarter. Target: minimum 4/quarter per primary channel
- **Messaging half-life:** How long before the winning message plateaus and needs re-testing? Track conversion trend over time post-winning rollout

## Integration Tips

- **HubSpot:** Create custom contact properties `messaging_variant_first_touch` and `messaging_variant_mql_trigger`. Build a report grouping MQL-to-opp rate and closed-won rate by variant. Use HubSpot's A/B email test feature for nurture sequence message testing.
- **Salesforce:** Add custom opportunity field `Message_Variant__c` (picklist). Build a pipeline dashboard filtered by this field to compare deal progression. Set up a Process Builder or Flow to auto-tag opportunities based on the campaign source that generated the lead.
- **Gong / Chorus:** Create trackers for each message variant's core phrase. Report weekly on: what percentage of calls include each messaging element? Correlate tracker presence in early calls with win rate at deal close.
- **Google Ads:** Use Ad Variations feature to run message experiments at scale without creating duplicate campaigns. Pin headlines by position (Headline 1 = value prop variant, Headline 2 = constant social proof element) for cleaner data.
- **LinkedIn Campaign Manager:** Use A/B testing feature for sponsored content. Create one campaign per message variant rather than multiple ads in one campaign to prevent LinkedIn's algorithm from self-optimizing before you have data.
- **Optimizely / VWO / Statsig:** For website tests, use the revenue impact calculator to set MDE before launching. Integrate with Salesforce to pass experiment assignment to CRM — this enables true revenue attribution rather than just conversion rate measurement.
- **Notion / Confluence:** Build a Messaging Experiment Log table with fields: Hypothesis, Channel, Start Date, End Date, Winner, Sample Size, Confidence Level, Revenue Impact, Key Insight, Next Test. This becomes your institutional messaging intelligence asset.

## Troubleshooting

**Problem: Tests reach statistical significance on CTR/open rate but I can't tell which message drives more pipeline or revenue.**
Solution: This is the most common failure mode — optimizing for a proxy metric instead of the real one. Fix: Tag every contact in your CRM with the message variant that generated them at first touch AND at MQL trigger. Set up a pipeline-by-source-variant report in HubSpot/Salesforce. Run the analysis after one full average sales cycle length has elapsed (not weekly). Accept that messaging experiments need 6-10 weeks minimum for revenue-level data — resist the pressure to call tests early on top-of-funnel metrics alone.

**Problem: Different messages win on different channels — I can't figure out what my "master" message should be.**
Solution: This is actually a signal, not a problem. Different messages winning on different channels usually means one of three things: (1) The audiences are different personas discovering you via different channels — you need ICP-specific messaging, not a universal message. (2) One message is better for awareness/interruption (top-of-funnel) and another for consideration/conversion (bottom-of-funnel) — both can be right for their stage. (3) Your winning "awareness" message is setting expectations your "conversion" message doesn't fulfill — you have a message-to-offer alignment problem. Run a message journey audit: map which message touches a buyer at each stage and check for cognitive dissonance.

**Problem: My traffic/sample sizes are too small — tests take 12+ weeks to reach significance.**
Solution: Prioritize testing on your highest-volume channel first (usually email if outbound-led, or ad copy if paid-led). Use Bayesian testing tools (Statsig, AB Smartly) which require smaller samples than frequentist methods for the same confidence level. For low-traffic channels, use "proxy" experiments: test messaging on paid search (can get 500+ impressions/day cheaply) and treat winning messages as validated hypotheses to roll out on lower-traffic channels without waiting for direct statistical significance on those channels. Alternatively, combine channels — if the same message wins on both email and LinkedIn ads simultaneously, treat that as cross-channel validation equivalent to single-channel significance at half the sample size per channel.
