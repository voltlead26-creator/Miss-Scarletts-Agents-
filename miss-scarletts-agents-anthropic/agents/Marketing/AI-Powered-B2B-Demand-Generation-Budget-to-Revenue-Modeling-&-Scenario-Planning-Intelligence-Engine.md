# AI-Powered B2B Demand Generation Budget-to-Revenue Modeling & Scenario Planning Intelligence Engine

**Difficulty:** Advanced | **Time:** 20-30 min | **Tags:** b2b, saas, analytics, revenue-forecasting, budget-planning, demand-generation, automation, strategy

## Overview

Builds a bottom-up demand generation revenue model that traces every marketing dollar from spend to MQL to SQL to pipeline to closed ARR — then runs bear/base/bull scenario simulations so CMOs can walk into any CFO conversation with defensible, data-grounded projections that hold up to scrutiny.

ANALYSIS FRAMEWORK — run all seven modules sequentially:

MODULE 1: BASELINE FUNNEL EFFICIENCY AUDIT
Before modeling forward, audit current funnel efficiency:

Step 1 — Conversion Rate Benchmarking:
Compare each stage conversion rate to B2B SaaS industry benchmarks:
- Lead → MQL: Company median 20-30%, Enterprise-focused median 15-22%
- MQL → SQL: Company median 13-20%, Enterprise-focused median 8-15%
- SQL → Opportunity: Company median 50-70%
- Opportunity → Closed Won: SMB median 20-30%, Mid-Market 18-25%, Enterprise 10-18%

Flag any stage converting >25% below benchmark as a [CRITICAL EFFICIENCY LEAK].

Step 2 — Channel Efficiency Matrix:
Rank all channels on a 2×2 matrix:
- X-axis: ARR Generated (low to high)
- Y-axis: Cost Per ARR (low to high, i.e., efficiency)
- Quadrant 1 (High ARR, Low Cost) = Scale Immediately
- Quadrant 2 (High ARR, High Cost) = Optimize Before Scaling
- Quadrant 3 (Low ARR, Low Cost) = Experiment Further
- Quadrant 4 (Low ARR, High Cost) = Cut or Restructure

Step 3 — Sales Capacity Alignment:
Calculate: (SQLs generated per month × SQL-to-Opp rate) ÷ Sales AE capacity per month = pipeline generation vs. sales consumption ratio. If >1.2x, marketing is outpacing sales (good problem). If <0.8x, marketing is the constraint.

MODULE 2: 12-MONTH BUDGET-TO-REVENUE MODEL (BASE CASE)
Build a month-by-month forecast table:

For each month (M1–M12), calculate:
- Marketing spend (total and by channel)
- MQLs generated (apply seasonal multipliers if provided)
- SQLs (MQLs × MQL→SQL conversion rate)
- Opportunities created (SQLs × SQL→Opp rate)
- Pipeline generated (Opps × ACV × 1.2 for average deal size variance)
- Projected ARR from this month's pipeline (accounting for sales cycle lag — i.e., M1 pipeline closes in M[1+sales cycle months])
- Cumulative ARR contribution vs. target

Key modeling rules:
- Apply a 3-month sales cycle lag between opportunity creation and revenue recognition
- NRR compounds monthly: existing ARR × (NRR/12) adds to total ARR each month
- Account for ramp time: new channels take 60-90 days to reach stated efficiency; model accordingly
- Apply quarterly seasonality multipliers if historical data shows patterns

MODULE 3: THREE-SCENARIO SIMULATION ENGINE

Run all three scenarios in parallel and present in a unified comparison table:

**BEAR CASE** (probability-weighted at 25%):

- MQL volume grows at current channel trajectory
- One conversion rate improves 5% each quarter (compounding effect)
- Result: projected ARR, months where target is exceeded/missed, required budget

**BULL CASE** (probability-weighted at 20%):
Triggers: New channel breakthrough, product-market fit acceleration, sales team expansion
- 20% budget increase in Q2
- 10% improvement in MQL→SQL rate (better qualification)
- 15% reduction in sales cycle (improved deal velocity)
- Addition of one new channel (model its ramp using historical new-channel data or use 60-day ramp assumption)
- Result: projected ARR, ARR upside vs. target, incremental investment required

## Troubleshooting

**Problem: The model produces an ARR number that seems too high/low compared to intuition**
*Cause:* Usually a data input error — particularly an incorrect win rate or ACV that doesn't reflect your actual book of business.
*Fix:* Cross-validate each funnel metric against three sources: (1) CRM report, (2) finance system, (3) quarterly board deck. If they differ, use the finance system number as the authoritative source for closed ARR and work backward to recalibrate the other rates. Add this instruction to the prompt: "Flag any metric that would require a >30% change in current conversion rates to hit target as mathematically implausible and explain the assumption gap."

**Problem: The CFO rejects the model because it doesn't account for deal slippage or seasonality**
*Cause:* A straight-line model without variance feels unrealistic to financially rigorous audiences.
*Fix:* Add the following modifier to the Advanced version input: "Apply a 12% quarterly deal slippage factor to all Opportunity→Closed ARR calculations. Apply these seasonality multipliers to MQL volume: Q1: 0.85x, Q2: 1.10x, Q3: 0.95x, Q4: 1.25x (or replace with your actuals)." This produces a more conservative, CFO-credible output without undermining the model's strategic case.

**Problem: Channel attribution data is unreliable — multiple channels touch the same deal**
*Cause:* Last-touch or first-touch attribution models distort channel efficiency rankings when buyers consume 6-12+ touchpoints before converting.
*Fix:* Add this instruction: "For channel efficiency ranking, use a blended attribution assumption: assign 40% of deal credit to first touch, 40% to last touch before opportunity creation, and 20% evenly distributed across middle touches. Note where this differs materially from last-touch data." This is a practical approximation that CFOs accept when a full multi-touch attribution model isn't available.
