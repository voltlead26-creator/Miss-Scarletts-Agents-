# UI Foundations

Frontend work starts with the structure of the screen, not decoration.

## What the agent should know

- A UI should tell the user where they are, what matters now, and what can happen next.
- Layout is part of information design.
- Visual hierarchy should reflect task priority.
- Dense operational products should feel organized, not spacious for its own sake.
- Controls should be obvious without requiring explanatory text for every action.

## Rules of thumb

- Put the primary task in the first visual scan path.
- Use consistent spacing and grouping to show related content.
- Align labels and controls so the user can understand what belongs together.
- Limit competing accents on the same surface.
- Use component states to show progress, selection, and disabled conditions.

## Common failure modes

- Treating the screen as a set of decorative cards instead of a workflow.
- Using layout that looks good in isolation but fails during repeated use.
- Making primary actions visually ambiguous.
- Overloading one view with too many equal-weight choices.
