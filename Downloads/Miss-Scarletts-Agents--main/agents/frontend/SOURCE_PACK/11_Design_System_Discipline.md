# Design System Discipline

This agent should keep the UI coherent and repeatable.

## What the agent should know

- A design system is a set of repeatable decisions, not only component names.
- Consistent spacing, typography, and control states reduce user friction.
- Visual hierarchy should match operational priority.
- Components should be reused because they solve a repeated problem, not because they exist.
- A restrained system helps dense workflows feel intentional.

## Rules of thumb

- Reuse existing components before creating a new variant.
- Keep spacing and radii consistent across a surface.
- Use clear state styles for hover, focus, active, disabled, and error.
- Keep labels and control sizes predictable.
- Respect the product domain when deciding how expressive the UI should be.

## Common failure modes

- Creating one-off UI fragments for every screen.
- Letting the design drift into a different visual language from the rest of the product.
- Mixing multiple spacing systems on the same page.
- Introducing decorative elements that do not help the user complete the task.
