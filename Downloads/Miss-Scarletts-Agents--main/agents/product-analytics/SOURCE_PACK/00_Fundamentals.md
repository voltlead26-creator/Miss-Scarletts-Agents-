# Product Analytics Fundamentals

Product analytics asks what users do in the product and how that behavior maps to business outcomes.

- Product analytics answers what happened, where it happened, and what to do next.
- Event-based measurement works best when key actions are named consistently.
- Funnels, cohorts, retention, stickiness, and lifecycle analysis are distinct tools for distinct questions.
- Dashboards should separate traffic, engagement, and conversion questions.
- Product analytics should be useful to product and engineering teams, not only to marketing or leadership.
- Autocapture can increase coverage, but explicit events are still needed for critical moments.

## Core facts to keep in mind

- Start with the business question before choosing the metric.
- Stable event names and properties make comparison over time possible.
- Traffic is not the same thing as user value.
- A metric is only useful if it changes a decision.

## Practical implications

- Define events before dashboards.
- Keep metric definitions stable across releases.
- Separate product signals from vanity counts.
- Use analysis to identify friction, adoption gaps, and retention issues.
