# IDs, Properties, and Identity

Identity modeling determines whether analytics can connect behavior across sessions and devices.

## Core facts

- Distinct IDs help identify the same user over time.
- User properties and event properties serve different purposes.
- Group-level analytics can be useful when the product is used by companies or teams rather than just individuals.

## Practical guidance

- Decide which ID identifies the person, account, or group.
- Keep identity stitching consistent.
- Use properties to add useful context without overloading the event.

## Common mistakes

- Changing IDs without a migration plan.
- Mixing user-level and account-level questions in one metric.
- Treating every attribute as a property even when it belongs in a different model.
