# Product Analytics Implementation Playbook

Use this when turning a product question into instrumentation or analysis.

1. State the business question.
2. Decide whether the answer is a metric, funnel, cohort, or dashboard.
3. Define the event names and properties.
4. Decide on the identity model.
5. Keep definitions stable enough for comparison.
6. Filter out irrelevant traffic or internal users when needed.
7. Review whether the metric actually drives a decision.

## Review questions

- What exact decision should this analysis support?
- Which event proves the behavior?
- Do we need a funnel, cohort, dashboard, or report?
- Are the definitions stable enough to compare over time?
- Is the metric a real proxy for user value?

## Output standard

- The question is answered directly.
- The measurement is trustworthy.
- The data model is clear.
- The result leads to an action.
