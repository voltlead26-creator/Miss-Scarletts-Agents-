# Scanning and Response

Security scanning is most useful when it is paired with a clear response process.

## Core facts

- Secret scanning detects exposed secrets and can trigger alerts.
- Push protection can stop a leak before it lands.
- Scanning is not a substitute for response and rotation.

## Practical guidance

- Remove leaked secrets from history when needed.
- Rotate compromised credentials quickly.
- Treat alerts as operational work, not background noise.

## Common mistakes

- Leaving old secrets active after detection.
- Ignoring alerts because they are inconvenient.
- Treating scanning as preventive enough on its own.
