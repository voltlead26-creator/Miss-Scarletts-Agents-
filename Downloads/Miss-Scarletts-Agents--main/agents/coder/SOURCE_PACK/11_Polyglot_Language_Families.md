# Polyglot Language Families

The Coder agent should be comfortable reasoning across major language families and file types.

## Scripting and application languages

- JavaScript and TypeScript for web and Node.js applications
- Python for automation, data work, and backend scripting
- Ruby and PHP for server-side web work
- Bash and PowerShell for shell automation and local tooling

## Compiled and systems languages

- Go for services, CLIs, and infra-adjacent tools
- Rust for performance-sensitive or safety-sensitive code
- Java and C# for enterprise applications and services
- C and C++ for lower-level integration and performance work

## Functional and multi-paradigm languages

- Kotlin and Swift for strongly typed application development
- F# and Elixir for expressive, concurrency-aware systems
- Haskell and other functional languages when the repo uses them

## Declarative and schema languages

- JSON, YAML, and TOML for config and metadata
- SQL for data access and schema changes
- HTML and CSS for document and UI structure
- GraphQL for typed APIs and client contracts

## Build and infrastructure languages

- Dockerfiles, Makefiles, shell scripts, and CI workflow YAML
- Terraform and similar IaC files

## Operating note

The exact syntax varies, but the coding questions stay the same: what does the code do, what can break, what must be tested, and what local convention should be preserved.
