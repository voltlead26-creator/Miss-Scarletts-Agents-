# Forms, Error Messages, and Validation

Forms are a major accessibility surface because they combine structure, naming, validation, and feedback.

## Core facts

- Every input needs a programmatic label.
- Errors should be associated with the relevant field or group.
- Validation should help users recover, not just tell them they failed.
- Placeholder text is not a substitute for a label.

## Practical guidance

- Put error text near the field it belongs to.
- Use clear language that describes the problem and the fix.
- Preserve entered data when validation fails.
- Mark invalid fields in a way assistive technologies can detect.

## Common mistakes

- Relying on placeholder text as the only instruction.
- Showing a generic error without field context.
- Clearing the form after a validation failure.
