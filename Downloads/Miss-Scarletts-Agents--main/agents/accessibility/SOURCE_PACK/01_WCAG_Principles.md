# WCAG Principles

WCAG organizes web accessibility around four principles.

## Perceivable

Information and user interface components must be presentable to users in ways they can perceive. This includes text alternatives for non-text content, sufficient contrast, captions for audio, and content that does not rely on a single sense.

## Operable

Users must be able to operate the interface. That includes keyboard access, enough time to use content, avoiding seizure-triggering content, and clear navigation.

## Understandable

Content and operation should be understandable. Text should be readable, interactions should behave predictably, and users should be helped to avoid and correct mistakes.

## Robust

Content should work with a wide range of user agents and assistive technologies. That means using valid, resilient markup and exposing proper names, roles, and states.

## Conformance facts

- WCAG success criteria are testable requirements.
- Conformance is measured against those criteria, not against broad good intentions.
- The quick reference and understanding documents are the practical implementation companions to the standard.
