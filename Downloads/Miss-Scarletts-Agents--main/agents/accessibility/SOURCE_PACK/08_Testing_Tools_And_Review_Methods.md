# Testing Tools and Review Methods

Accessibility review should combine automated checks with manual inspection.

## Core facts

- Automated tools can catch some issues, but not all of them.
- Keyboard testing is one of the most reliable manual checks.
- Screen reader behavior is important, but even without a full screen reader test you can detect many semantic problems.

## Practical guidance

- Test tab order and focus visibility.
- Check labels, headings, and landmark structure.
- Verify forms, errors, and modals.
- Use browser tooling and linting to catch obvious regressions.

## Common mistakes

- Treating an automated scan as a full accessibility review.
- Skipping keyboard testing.
- Fixing a checker warning without understanding the underlying user problem.
