# Accessible Writing

Accessible documentation is usable by more people because it respects how people read and process information.

## Core facts

- Accessible documentation helps readers with disabilities and improves comprehension for everyone.
- Alt text should be concise and descriptive.
- Error messages and documentation should not rely only on color or visual cues.

## Practical guidance

- Write directly and plainly.
- Make structure visible through headings and lists.
- Describe images and diagrams in a way that matches their purpose.

## Common mistakes

- Using decorative language where clarity matters.
- Leaving visual content unexplained.
- Writing in a way that depends on one mode of perception.
