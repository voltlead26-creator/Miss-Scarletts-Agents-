# Format and Tone

Formatting and tone are part of comprehension.

## Core facts

- Error messages should use a positive, instructive tone.
- Formatting should make the important part easy to find.
- The same problem should produce the same message.

## Practical guidance

- Keep tone neutral and useful.
- Use consistent formatting for related messages.
- Use progressive disclosure when the explanation is long.

## Common mistakes

- Blaming the user.
- Excessive apology or humor in a failure path.
- Letting similar messages diverge in format and meaning.
