# Error Messages

Error messages are a special form of technical writing because they must explain a problem and the next action quickly.

## Core facts

- Good error messages answer what went wrong and how to fix it.
- Error messages should identify the cause, invalid input, or constraint when possible.
- Error messages should be concise, positive in tone, and aimed at the target audience.

## Practical guidance

- Name the invalid value or condition.
- Explain the expected format or requirement.
- Provide an example when it helps the reader recover.

## Common mistakes

- Failing silently.
- Using generic errors that do not identify the cause.
- Being overly apologetic or vague.
