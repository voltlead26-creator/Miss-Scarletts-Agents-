# Documentation Maintenance

Documentation is not finished when it ships. It needs maintenance as the product changes.

## Core facts

- Living documentation should stay aligned with the product and the code.
- Old examples and broken links degrade trust quickly.
- Maintenance is part of documentation quality.

## Practical guidance

- Update docs when behavior changes.
- Remove outdated examples and broken references.
- Keep the doc set internally consistent.

## Common mistakes

- Publishing once and never revisiting the content.
- Letting docs drift from the product.
- Leaving placeholder text in a shipped document.
