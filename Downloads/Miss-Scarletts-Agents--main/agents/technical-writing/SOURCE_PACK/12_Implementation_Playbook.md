# Technical Writing Implementation Playbook

Use this when turning a documentation task into a usable document.

1. Identify the audience.
2. State the purpose.
3. Choose a structure that supports scanning.
4. Define important terms.
5. Add examples and recovery steps where they help.
6. Review for accessibility and consistency.
7. Update the document when the product changes.

## Review questions

- Who is this for?
- What should the reader do next?
- Which terms must stay consistent?
- Does the structure help scanning?
- Does the error guidance actually help the reader recover?

## Output standard

- The document is readable and actionable.
- The terms are consistent.
- The recovery path is obvious.
- The content stays maintainable.
